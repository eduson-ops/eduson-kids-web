😰 Проблема: повторение кода

Ты хочешь спавнить врага в 3 разных местах. Без функции — так:

❌ Без функции — 18 строк
@старт:
    переменная враг1 = создать_объект(
        "зомби", [10, 0, 10])
    враг1.здоровье = 100
    враг1.скорость = 2

    переменная враг2 = создать_объект(
        "зомби", [-10, 0, 10])
    враг2.здоровье = 100
    враг2.скорость = 2

    переменная враг3 = создать_объект(
        "зомби", [0, 0, -15])
    враг3.здоровье = 100
    враг3.скорость = 2
✅ С функцией — 6 строк
функция спавн_врага(x, z):
    переменная е = создать_объект(
        "зомби", [x, 0, z])
    е.здоровье = 100
    е.скорость = 2
    вернуть е

@старт:
    спавн_врага(10, 10)
    спавн_врага(-10, 10)
    спавн_врага(0, -15)

💡 Принцип DRY — Don't Repeat Yourself («не повторяй себя»). Функция — способ упаковать повторяющееся действие в одну команду.

📡 Собственные события

KubiK позволяет создавать свои события — функции-обработчики с декоратором @.

Встроенные события
@старт
@нажата_клавиша( пробел )
@столкновение( объект )
@каждые( 3 сек )
@клик_мыши( кнопка )
Комбинации — мощь!
мир счёт = 0

@каждые(5, сек):
    спавн_врага(
        случайное(-50,50),
        случайное(-50,50)
    )

@столкновение(монета):
    добавить_очки(10)
    удалить(монета)

@нажата_клавиша(пробел):
    игрок.прыгнуть(высота=6)

функция добавить_очки(n):
    счёт += n
    сказать(текст(счёт))

🎛️ Параметры vs аргументы

Терминология важна — она используется в Python, JS, любом языке.

Параметр — при объявлении
функция спавн(x, z):
              ↑    ↑
         параметры (имена-заглушки)
Параметр — это «ящик» внутри функции. Пока функция не вызвана, в нём пусто.
Аргумент — при вызове
спавн(10, -5)
         ↑     ↑
    аргументы (реальные значения)
Аргумент — реальное значение, которое подставляется в «ящик» при вызове.
Визуализация передачи
вызов
спавн(10, -5)
→
внутри функции
x = 10, z = -5
→
результат
зомби на [10,0,-5]

🛠️ Задание: «Волны врагов»

Сделай систему волн: с каждой волной враги становятся сильнее.

1️⃣
Объяви переменные волны
мир волна = 1
мир живые_враги = 0
2️⃣
Напиши функцию спавна
функция начать_волну(номер)
3️⃣
Вызывай при победе
если живые_враги == 0
начать_волну(волна + 1)
мир волна = 1
мир живые_враги = 0

функция начать_волну(номер):
    волна = номер
    переменная кол = номер * 3
    повторить кол раз:
        переменная е = создать_объект(
            "зомби",
            [случайное(-40,40),0,
             случайное(-40,40)])
        е.здоровье = 50 * номер
        живые_враги += 1

@старт:
    начать_волну(1)

@столкновение(пуля):
    живые_враги -= 1
    если живые_враги <= 0:
        сказать("Волна " + текст(волна) +
                " пройдена!")
        начать_волну(волна + 1)

⚡ Функции без «вернуть»

Не все функции возвращают значение. Некоторые просто делают — меняют мир, показывают текст.

Функция-действие (void)
функция показать_hud():
    сказать(соединить(
        "♥ ", текст(жизни),
        "  ⭐ ", текст(счёт)
    ))

# Вызов — просто имя без присваивания
@каждые(2, сек):
    показать_hud()
Нет вернуть → ничего не возвращает. Просто выполняет действие.
Функция-вычисление (return)
функция итоговый_урон(база, броня):
    переменная урон = база - броня
    если урон < 1:
        урон = 1
    вернуть урон

# Вызов — результат используем
@столкновение(враг):
    переменная д = итоговый_урон(30, 10)
    жизни -= д
    сказать(соединить("-", текст(д), " HP"))

⚡ Правило: если результат нужен снаружи — вернуть. Если только делаем что-то — без возврата.

🐍 Функции в Python

Нажми переключатель Python в редакторе — KubiK покажет тот же код на Python.

KubiK Text
функция начать_волну(номер):
    волна = номер
    переменная кол = номер * 3
    повторить кол раз:
        спавн_врага(
            случайное(-40,40),
            случайное(-40,40),
            50 * номер
        )

@каждые(5, сек):
    если живые_враги <= 0:
        начать_волну(волна + 1)
Python 3 (автогенерация)
def начать_волну(номер):
    global волна
    волна = номер
    кол = номер * 3
    for _ in range(кол):
        спавн_врага(
            random.randint(-40, 40),
            random.randint(-40, 40),
            50 * номер
        )

@every(5, "sec")
def _():
    if живые_враги <= 0:
        начать_волну(волна + 1)

Видишь разницу? def вместо функция, global для мировых переменных, range() вместо повторить...раз. Логика — та же.

🐛 Ошибки в функциях

E-KB-0031: неверное число аргументов
функция спавн(x, z):   # ожидает 2 аргумента
    ...

спавн(10)              # ❌ передан 1 аргумент
спавн(10, 5, 2)        # ❌ передано 3 аргумента
спавн(10, -5)          # ✅
⚠️ Рекурсия без выхода (бесконечный вызов)
функция бесконечно():
    бесконечно()    # ❌ вызывает себя без условия выхода
    # E-KB-0055: глубина вызовов > 100
функция обратный_отсчёт(n):
    если n <= 0:
        сказать "Старт! 🚀"
        вернуть         # ← условие выхода
    сказать(текст(n))
    обратный_отсчёт(n - 1)  # ✅ с уменьшением

📝 Проверка знаний

Что выведет код?
функция дважды(x): вернуть x * 2
сказать(дважды(7))

✅ Верно! дважды(7) → 7 * 2 → 14. Функция возвращает число, сказать выводит 14.
