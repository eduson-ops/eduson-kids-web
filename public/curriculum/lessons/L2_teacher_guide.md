# Методичка учителю · Урок 2 · Движение персонажа

## Подготовка (5 минут)

- [ ] Проверить что у всех учеников открывается их проект `L1-obby-{имя}`
- [ ] Открыть `L2_player_movement.html` на проекторе
- [ ] Подготовить резервный проект `eduson://template/L1-obby-ready` на случай если кто-то потерял L1
- [ ] Раскрасить в блокли-палитре мелом 3 цвета (жёлтый/синий/фиолетовый) — помогает визуально запомнить категории блоков

## Цели урока

После L2 ученик может:

1. **Отличить** Player Script от Object Script.
2. **Собрать** цепочку из 2-3 блоков (событие → действие).
3. **Использовать** переменную-счётчик (`прыжки`).
4. **Связать** действие с клавишей.
5. **Отладить** свой скрипт (если не работает — прочитать блоки сверху вниз).

CS-концепты: **скрипт, событие, действие, переменная (введение)**.

## Таймлайн (45 минут)

| Время | Блок | Учитель | Ученик |
|---|---|---|---|
| 0-5 | Welcome + повторение | Слайды 1-2 | Отвечает что такое Part/Spawn |
| 5-15 | Теория блоков | Слайды 3-4 | Задаёт вопросы |
| 15-20 | Демо-скрипт | Слайды 5-6, вживую собирает спринт | Смотрит |
| 20-40 | Практика | Слайды 7-9, помогает по кабинету | Собирает 3 скрипта |
| 40-45 | Квиз + домашка | Слайды 10-12 | Отвечает, записывает |

### 0-5 мин · Welcome

Опрос (быстрый, по цепочке):

- "Что такое Part?" — 3D объект
- "Зачем Anchor?" — не падать
- "Что делает Spawn?" — появляет игрока

Если кто-то забыл — не ругаем, напоминаем за 10 секунд.

### 5-15 мин · Теория блоков

Важно **не перегружать**. Три цвета блоков — запоминаем.

- **Жёлтые** = события ("КОГДА")
- **Синие** = действия ("ЧТО СДЕЛАТЬ")
- **Фиолетовые** = условия ("ЕСЛИ") — пока мельком

На слайде 10 есть Python — **показать, но не писать**. Скажи: "На уроке 4 увидите как это выглядит буквами. Пока — только блоки".

### 15-20 мин · Демонстрация

В живом редакторе на проекторе собрать спринт (слайд 7). Обязательно:

- Показать **где** лежит "Event → KeyPress"
- Показать **зачем** ставить "скорость = 12" при отпускании (иначе останется 24 навсегда)

### 20-40 мин · Практика

Дети делают 3 скрипта в одном Player Script:

1. Спринт (5 мин)
2. Двойной прыжок (10 мин) — самое сложное
3. Респавн на R (5 мин)

**Чек после каждого блока** — прошёл ли по цепочке, запустилась ли игра.

### 40-45 мин · Wrap-up

Квиз (слайд 11), домашка (слайд 12).

## Частые ошибки

| Ошибка | Симптом | Решение |
|---|---|---|
| Блоки не "защёлкиваются" | Висят отдельно | Подтянуть к верхнему, дождаться магнита |
| Нет блока "KeyPress" | Категория пустая | Убедиться что выбран Player Script, не Object |
| Спринт остаётся навсегда | Игрок бежит и без Shift | Забыл "когда отпущена Shift → скорость = 12" |
| Прыжок срабатывает один раз | Двойной не работает | `прыжки = 0` стоит НЕ в событии "приземлился" |
| R не телепортирует | Ничего не происходит | Spawn должен быть переменной-ссылкой; перетащить объект Spawn из сцены в блок |
| Путаница Player vs Object | Скрипт на платформе не слышит клавиши | Player Script привязывается к Spawn, а не к платформе |

## Дополнительные задачи для продвинутых

1. **Инвертированное управление.** Клавиша <kbd>I</kbd> меняет WASD местами — W вниз, S вверх.
2. **Dash.** Двойной <kbd>A</kbd> — рывок влево на 10 юнитов.
3. **Гравитация-переключатель.** Клавиша <kbd>G</kbd> меняет гравитацию Earth ↔ Moon.
4. **Тихий режим.** Shift+C отключает звук прыжка (готовит к L3).

## Мостик к L3

Сегодня мы делали события **от клавиатуры**. На L3 будут события **от столкновений** — игрок коснулся монетки → +1 очко. Это более "игровое" событие.
