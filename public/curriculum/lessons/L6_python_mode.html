⚠️ Главный принцип Python: отступы (пробелы) задают структуру кода. Забыл отступ — ошибка.

⬜ Отступы — сердце Python

В Python нет скобок для блоков. Структуру задают пробелы. Это главное отличие от большинства языков.

❌ Неверные отступы
def проверить(x):
print("начало")  # ← нет отступа!
    if x > 0:
return x        # ← неверный уровень!

# IndentationError: ожидался отступ
✅ Правильные отступы
def проверить(x):
    print("начало")   # 4 пробела
    if x > 0:
        return x      # 8 пробелов (вложен)
    return 0          # 4 пробела
Уровни вложения:
def волна(номер):
for i in range(номер):
if i % 2 == 0:
спавн(i * 10, 0)
Каждый уровень = 4 пробела. KubiK editor расставляет их автоматически.

⚠️ Правило global в Python

Если ты изменяешь переменную, объявленную вне функции, — напиши global имя_переменной в начале функции. Иначе Python создаст новую локальную переменную.

score = 0

def wrong():
    score += 1     # ❌ UnboundLocalError!

def correct():
    global score
    score += 1     # ✅

🎮 Полная игра на Python

Вот полный скрипт «Сборщика монет» на чистом Python в KubiK-редакторе:

# Eduson Kids — KubiK Python mode
# Урок 6: Сборщик монет

# ── Мировые переменные ──
score = 0
lives = 3
record = 0
wave = 1

# ── Вспомогательные функции ──
def add_score(points):
    global score, record
    score += points
    if score > record:
        record = score

def lose_life(damage=1):
    global lives
    lives -= damage
    say(f"♥ {lives}")
    if lives <= 0:
        say("Игра окончена 💀")

# ── Обработчики событий ──
@on_start
def _():
    say("Собирай монеты! 🪙")

@on_collision("coin")
def _(coin):
    add_score(10)
    remove(coin)
    say(f"Счёт: {score} ⭐")
    if score >= 100:
        say("Победа! 🎉")

@on_collision("spike")
def _():
    lose_life(1)

@on_key("space")
def _():
    player.jump(height=6)

🔍 Отладка: print() и say()

Три инструмента отладки в Python-режиме KubiK.

1. say() — выводит в игровой чат
@on_collision("coin"):
    add_score(10)
    say(f"DEBUG: score={score}, lives={lives}")
Видно в игре — хорошо для тестирования логики.
2. print() — в консоль редактора
def add_score(points):
    global score
    score += points
    print(f"[DEBUG] score: {score}")  # только в консоли IDE
3. assert — проверка инвариантов
def lose_life(damage):
    global lives
    assert damage > 0, "Урон должен быть положительным"
    lives -= damage
Если assert ложный — игра останавливается с понятным сообщением.
