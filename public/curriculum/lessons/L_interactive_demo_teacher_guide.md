# Методичка · Тестовый урок «Живой режим»
*Для преподавателя · Эдюсон Kids Eduson Kids Platform · демонстрация ключевой фишки «блок управляет миром»*

---

## Краткая карта урока

| Параметр | Значение |
|---|---|
| Длительность | 45 мин |
| Возраст | 10–15 лет |
| Тип | Демо-урок (мост M1↔M2) |
| Требуется | Studio открыта в 3-табовом режиме · вкладка «Скрипт» в L1 «Блоки» |
| Файл презентации | `L_interactive_demo.html` |

## Цели урока

По завершении ученик умеет:

1. Включить режим **⚡ Живой** в Studio-Script.
2. Перетащить блок и **увидеть мгновенно**, как сцена меняется во вкладке Test (без кнопки «Запустить»).
3. Объяснить цикл: *блок → Python → команда миру → изменение сцены*.
4. Собрать мини-игру из 8 блоков, увидеть её в действии в реальном времени.
5. Выключить живой режим когда скрипт становится сложным — и понимать **зачем**.

## Подготовка

- Studio открыта в `/studio`.
- Во вкладке «🧩 Скрипт» переключено на режим **L2 «Блоки + Python»** (мост). Блоки слева, Python справа — ребёнок видит оба одновременно.
- В верхнем тулбаре — чекбокс **⚡ Живой режим**. Пока выключен.

## Урок по 9-шаговому шаблону

### Шаг 1 · Организационный момент (3 мин)

Приветствие. «Помните, мы рассматривали события? `@старт`, `@столкновение`. Что ещё они умеют?» Пара ответов (словарь L3 активирован).

### Шаг 2 · Повторение через игру (7 мин)

**Мини-quiz** (из `L_interactive_demo.html`):

1. Чем *блок* отличается от *обычной команды в коде*? → Блок — это визуальная версия одной строки кода.
2. Что делает блок `📍 поставить блок красный`? → Создаёт в мире красный Эдюсон Kids.
3. Мост-вопрос: **«Если я перетащу блок — надо ли каждый раз нажимать Запуск, чтобы увидеть результат?»** → Пока ответ «да», но сегодня научимся «нет».

### Шаг 3 · Концепция + аналогия из реального мира (7 мин)

**Аналогия — радиоуправляемая машинка.** Учитель объясняет:

> «Представьте машинку на пульте. Вы крутите руль — машинка поворачивает *сразу*. Не нужно нажимать «Поехали», потом «Поверни». Блоки — это руль. Вкладка Test — это машинка. Сегодня мы включим радио.»

**Словарь урока:**

| Термин | Определение | Аналогия |
|---|---|---|
| **Живой режим** | Редактор сам запускает код при каждом изменении блоков | Автосохранение в документе |
| **Blocks-to-Code** | Превращение блоков в Python-строки | Перевод с картинок на буквы |
| **Команда миру** | Одна строка кода = одна инструкция игре | Приказ солдату — «иди!», «прыгни!» |
| **Debounce** | Задержка 500мс: меняешь много блоков подряд — прогон один | Не дёргаешь лифт 10 раз — он приедет один раз |

Дополни: «Debounce — чтобы не заваливать систему, когда мы быстро перетаскиваем много блоков».

### Шаг 4 · Учитель моделирует вслух (8 мин)

Учитель на большом экране демонстрирует:

1. Режим **L2 «Мост»**. Блоки слева, Python справа.
2. Включает **⚡ Живой режим** — чекбокс вверху. Появляется пульсирующий **ON**-значок.
3. Думает-вслух:

   > «Смотрите: я перетаскиваю блок `🧱 поставить блок` сюда, в `@старт`. Цвет — красный. Координаты — (3, 0, 0). И СМОТРИТЕ на экран справа…»

4. Переключается на вкладку **▶ Тест**. В сцене появляется красный Эдюсон Kids.
5. Возвращается в **🧩 Скрипт**. Меняет цвет блока на зелёный.
6. Снова Test — Эдюсон Kids стал зелёным.

**Ключевой момент:** учитель делает паузу и говорит вслух: «Я не нажимал Запуск. Ни разу. Система сама заметила, что блок изменился, превратила его в Python-код, отправила команду в мир — и мир откликнулся.»

### Шаг 5 · Пошаговая практика по инструкции (12 мин)

Ученики на своих машинах повторяют:

1. Открой Studio → Скрипт → включи **⚡ Живой режим**.
2. Построй программу из этих блоков (в `@старт`):

   ```
   повторить 5 раз:
       поставить блок жёлтый
       идти вперёд 2 шага
       повернуть направо
   ```

3. Открой Test. Наблюдай: должна появиться «лесенка» из 5 жёлтых Эдюсон Kidsов.
4. Попробуй на лету:
   - Поменять цвет → **мгновенно** все Эдюсон Kidsи перекрасятся.
   - Поменять число повторов с 5 на 10 → через 0.5с появится 10 Эдюсон Kidsов.
   - Поменять `идти вперёд 2 шага` на `идти вперёд 4 шага` → геометрия растянется.

Учитель ходит, смотрит экраны, ловит «вау-реакции» и типичные затыки.

### Шаг 6 · Групповой разбор ошибок (3 мин)

Частые проблемы (разобрать на экране):

- «Ничего не появляется» → проверь галочку Живой режим, проверь что `@старт` есть, что блоки *внутри* него.
- «Появилось, но странно» → часть блоков повторяется от прошлых запусков; нажми «Сброс» в Build-табе.
- «Тормозит» → сложный цикл, выключи Живой режим, используй кнопку «Запустить» вручную.

**Нормализация:** «Это не баги — это *ожидаемое поведение*. Живой режим — как автопилот: удобно на трассе, но в пробке лучше руками.»

### Шаг 7 · Перемена

Встроена в школьный режим.

### Шаг 8 · Открытая задача (15 мин дома / 5 мин в классе)

**Задание:** «Сделай **интерактивную вывеску**. Если поменять одну переменную — вся вывеска перекрасится и переедет».

Дифференциация:

- **🟢 10–11 лет** · Постройте 3 Эдюсон Kidsа в ряд. Всё живое. Добавьте монетку сверху.
- **🟡 11–13 лет** · **Функция** `постройка(цвет)` из 5 блоков. Вызовите её 3 раза с разными цветами. Измените функцию — все три вызова переразбираются автоматически.
- **🔴 13–15 лет** · **Перейдите в L3 Python**. Откройте Monaco-редактор. Напишите цикл `for` с параметрами. Проверьте, что **Python в Live-режиме тоже запускается автоматически** на каждое нажатие клавиши (через дебаунс 500мс). Это прямое окно в то, как работают современные IDE (VS Code с hot-reload).

### Шаг 9 · Итог + предвкушение (2 мин)

Каждый ребёнок вслух: **«Я включил Живой режим и увидел как мой блок ___ создал ___ в мире»**.

Тизер: «На следующем уроке мы начнём читать настоящий Python. Вы уже увидели, как блоки превращаются в код — а теперь научимся писать такой код своими руками.»

## Частые ошибки

| Ошибка | Причина | Решение |
|---|---|---|
| Живой режим включён, но мир не обновляется | Цикл в блоке с ошибкой — Python не сгенерировался | Проверить «красные» блоки, исправить |
| Мир захламляется лишними Эдюсон Kidsами | Каждый запуск добавляет новые; старые не удаляются | «Сброс» в Build-табе. (В v1.0 добавим авто-очистку при live-режиме.) |
| Браузер начал лагать | Слишком много блоков + много команд = тысячи `place_block` за раз | Выключить Живой режим, уменьшить число повторений |
| Пульсирует ON, но живого эффекта нет | Вкладка Test закрыта | Переключись на ▶ Тест — эффект только там |

## Для учителя: почему это важно

**Immediate feedback — один из самых мощных педагогических драйверов.** Papert в «Mindstorms» писал: «Дети учатся, когда видят прямую связь между своим действием и результатом. Задержка убивает любопытство.» Наша реализация (debounce 500мс, step 30мс в Test) — технологически воспроизводит этот принцип.

**PRIMM — Predict/Run/Investigate/Modify/Make.** Этот урок — идеальный *Modify*-цикл: ребёнок модифицирует блок, мгновенно видит следствие, строит модель. Раньше модификация требовала «нажми Запуск» — это тормозило.

**Сигнал к переходу в Python.** Когда ребёнок в Live-режиме начинает экспериментировать с параметрами — это значит, мы готовы к L6 «Python-мост». У нас появляется ощущение *живого* кода, а не статичного текста.

**Ограничения:**
- Цикл с большим числом итераций (>500) в Live-режиме может затормозить. Объясните это открыто — это компромисс.
- Нет автоочистки сцены между прогонами. В v1.1 добавим «чистую песочницу» как опцию.
