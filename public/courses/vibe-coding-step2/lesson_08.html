DNS / A-record

Как работает DNS

Браузер не знает IP-адресов. Он знает только человеческие имена. DNS — это система, которая переводит одно в другое.

05 / 22 — HTTPS: шифрование трафика
TLS / HTTPS

HTTPS = HTTP + шифрование

HTTP

Открытый текст

Весь трафик виден любому между тобой и сервером: Wi-Fi роутеру кафешки, провайдеру, школьной сети. Логины, пароли, токены — всё читается.

POST /login HTTP/1.1
Host: site.ru

login=vanya&password=12345  ← видно всем
HTTPS

Зашифрованный канал

TLS-рукопожатие создаёт приватный канал. Даже если перехватят — увидят только мусор. Браузер показывает зелёный замочек.

POST /login HTTP/1.1
Host: site.ru

●◉◒◐◑◓▲◊⬢⬡◈◎◯◌  ← зашифровано

TLS — это преемник SSL. Имена используют как синонимы, но технически сейчас везде TLS 1.2 и TLS 1.3. «SSL-сертификат» = «TLS-сертификат», просто название прижилось.

06 / 22 — Let's Encrypt
Certificate Authority

Let's Encrypt — бесплатный CA

Центр сертификации (Certificate Authority, CA) — это организация, которой доверяют все браузеры. Она выпускает сертификаты, подтверждающие: «да, это действительно тот сайт, за который себя выдаёт».

Раньше: сертификат стоил 2000–20 000 ₽/год у Comodo, DigiCert, GlobalSign.

С 2015: Let's Encrypt выдаёт то же самое бесплатно и автоматически. Спонсоры — Mozilla, Cisco, EFF, Google.

Характеристики

Цена0 ₽
Срок действия90 дней
Автопродлениеcertbot --nginx

Wildcards*.your.ru — есть
ШифрыTLS 1.2, 1.3

09 / 22 — Шаг 2: A-запись
Step 02

Привязываем домен к IP твоей VM

В личном кабинете reg.ru находим раздел «DNS» / «Управление зоной». Добавляем две A-записи:

ХостТипЗначениеTTLСмысл
@A185.23.55.103600корень your.ru
wwwA185.23.55.103600www.your.ru
apiA185.23.55.113600поддомен (опционально)

Проверяем распространение DNS

# На своём компе:
ping your.ru
# должен вернуть твой IP VM

# Или через dig:
dig your.ru +short
nslookup your.ru

DNS обновляется не мгновенно. Обычно 5–20 минут, иногда до 24 часов. Если через 5 минут не работает — не паникуй, сделай чай и подожди 20.

10 / 22 — Шаг 3: установка certbot
Step 03

Ставим certbot на VM

Заходим по SSH на свою VM и ставим certbot с плагином для Nginx:

# Обновляем список пакетов
sudo apt update

# Ставим certbot и nginx-плагин
sudo apt install certbot python3-certbot-nginx -y

# Проверяем, что встало
certbot --version
# → certbot 2.9.0

Что делает nginx-плагин?

A

Автонастройка

Сам редактирует Nginx-конфиг — добавляет listen 443 и пути к сертификату.

B

HTTP-01 challenge

Let's Encrypt проверяет, что ты владеешь доменом, через файл на порту 80.

C

Редирект 80 → 443

Можно включить одной командой — весь трафик форсированно идёт по HTTPS.

12 / 22 — Nginx-конфиг после certbot
Config diff

Что certbot допишет в конфиг

server {
    server_name your.ru www.your.ru;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # ← Добавлено certbot:
    listen 443 ssl;
    ssl_certificate     /etc/letsencrypt/live/your.ru/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your.ru/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;
}

server {
    # Редирект HTTP → HTTPS
    if ($host = your.ru)     { return 301 https://$host$request_uri; }
    if ($host = www.your.ru) { return 301 https://$host$request_uri; }
    listen 80;
    server_name your.ru www.your.ru;
    return 404;
}

14 / 22 — Автопродление
Renewal

Сертификат живёт 90 дней — что дальше?

Let's Encrypt выдаёт короткие сертификаты специально: чтобы заставить всех автоматизировать продление. Certbot делает это за тебя — через systemd timer или cron.

Проверяем, что автопродление работает:

# Dry-run — безопасная проверка
sudo certbot renew --dry-run

# Успех:
# Congratulations, all simulated renewals
# succeeded!

# Посмотреть таймер systemd:
systemctl list-timers | grep certbot

Как это работает

  • Каждый день systemd-таймер запускает certbot renew
  • Certbot проверяет: до истечения < 30 дней?
  • Если да — продлевает, обновляет файлы
  • Перезагружает Nginx (systemctl reload nginx)
  • Ты даже не замечаешь — всё произошло в 3 часа ночи

Настроил один раз — забыл на годы. Certbot сам пришлёт email, если что-то пойдёт не так.

16 / 22 — Если что-то пошло не так
Troubleshooting

Типичные ошибки и решения

!

A-запись не подтянулась

DNS не успел распространиться. Ждём до 24 часов, обычно 5–10 минут.

dig your.ru
nslookup your.ru
!

Certbot: rate limit

Не запускай 10 раз подряд. Лимит — 5 успешных запусков в неделю на домен.

Используй --staging для тестов.

!

Swagger битый через HTTPS

В FastAPI ставим root_path или проверяем proxy-заголовки в Nginx.

!

80 порт закрыт firewall'ом

Certbot общается с LE через 80. sudo ufw allow 80 — и запускаем снова.

!

Mixed content warning

Страница HTTPS, но картинка подгружается по HTTP. Правим в коде или через CSP.

!

DNSSEC mismatch

Если включил DNSSEC на reg.ru — отключи для школьного проекта. Причина половины странных ошибок.

17 / 22 — Проверяем себя через SSL Labs
Quality check

Получаем grade A+

SSL Labs от Qualys — бесплатный инструмент от индустрии безопасности. Прогоняет твой HTTPS через десятки тестов и ставит оценку.

Открой: https://www.ssllabs.com/ssltest/ — и вбей свой домен.

Шкала оценок

  • A+ Отлично — HSTS, современные шифры
  • A Хорошо — всё правильно, но без HSTS
  • B Приемлемо — есть к чему стремиться
  • C / F Проблемы с шифрами или цепочкой

Что прокачать до A+

# В Nginx добавляем HSTS:
add_header Strict-Transport-Security
  "max-age=31536000;
   includeSubDomains;
   preload" always;

# Запрет старых протоколов
ssl_protocols TLSv1.2 TLSv1.3;

# OCSP stapling
ssl_stapling on;
ssl_stapling_verify on;

HSTS говорит браузеру: «этот сайт только по HTTPS, забудь про HTTP навсегда».
