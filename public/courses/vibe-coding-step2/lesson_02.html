Теория 05 / 22

Анатомия HTTP-запроса

Method GET
URL https://api.openweathermap.org/data/2.5/weather
Query q=Moscow&appid=xxx
Headers Accept: application/json
User-Agent: vibe-step2/1.0
Body — (GET не имеет тела)
  • Метод — что я хочу сделать: забрать, отправить, обновить, удалить
  • URL — куда: домен + путь к ресурсу
  • Query — уточнения через ? и &
  • Headers — надпись на конверте: токен, формат, язык
  • Body — письмо внутри конверта (только для POST/PUT)
HTTP-методы 06 / 22

Четыре метода — почти весь REST

GET

Забрать

Получить данные. Без тела. Безопасно можно дёргать тысячи раз.

/users, /posts/42

POST

Создать

Отправить данные, создать новую сущность. Тело обязательно.

/users, /orders

PUT

Обновить

Заменить сущность целиком. Два вызова = один результат.

/users/42

DELETE

Удалить

Снести ресурс. Удалить удалённое — ничего страшного.

/orders/7

CRUD = Create (POST) · Read (GET) · Update (PUT) · Delete (DELETE)

Статус-коды 07 / 22

Статус — это код для машины, а не ошибка

Код, по которому ты понимаешь, что делать дальше. Три семейства — и ты владеешь 95% случаев.

2xx

Успех

200 OK — данные пришли

201 Created — создано

204 No Content — ок, но без тела

«Всё ок.»
4xx

Клиент облажался

400 Bad Request — кривой запрос

401 Unauthorized — кто ты?

403 Forbidden — тебе сюда нельзя

404 Not Found — адреса нет

429 Too Many — притормози

«Ты облажался.»
5xx

Сервер облажался

500 Internal Error — упал сервер

502 Bad Gateway — прокси сломался

503 Unavailable — сервер перегружен

«Сервер облажался.»
REST 08 / 22

REST — это не язык, это стиль

Набор договорённостей, как именно строить API. Придумал Рой Филдинг в 2000 году. До него все городили SOAP — это было ужасно многословно.

❌ Не REST

POST /getUserById
Content-Type: application/json

{"userId": 42}

Глаголы в URL, POST для чтения, кастомные методы — всё как в 1998-м.

✓ REST

GET /users/42
Accept: application/json

Ресурс в URL, метод GET говорит «я читаю», тело не нужно. Кратко и читаемо.

«Если ты можешь прочитать ответ API, ты можешь написать бота, который его дёргает.»

JSON 09 / 22

JSON — таблица в Excel, только в тексте

Пример ответа API

{
  "temp": -4,
  "city": "Moscow",
  "wind": {
    "speed": 5.2,
    "deg": 220
  },
  "weather": ["snow", "cold"],
  "ok": true
}
json

Что тут есть

  • Объект — всё в { }
  • Ключ: значение — через двоеточие
  • Числа — без кавычек
  • Строки — в двойных кавычках
  • Массив — в [ ]
  • Вложенные объекты — сколько угодно
  • true / false / null — без кавычек

JSON парсится любым языком за одну строку. Меньше XML в 2–3 раза.

Инструмент 11 / 22

Почему Bruno — а не Postman

01 Open-source. Бесплатный, форкается на GitHub, не продаёт твою коллекцию.
02 Коллекции в файлах. Каждый запрос — это .bru-файл. Можно положить в git commit.
03 Без регистрации. Скачал — запустил. Никаких аккаунтов через почту родителей.
04 Работает без VPN. Скачивается напрямую, не блокируется РКН.
05 Environments + Variables. Один раз сохранил ключ — дёргаешь через {{openweather_key}}.
06 Diff в PR. Если работаешь в команде — изменения в коллекции видны в pull request.

«Postman переехал в облако в 2024-м и стал закрытым. Bruno — это Postman, каким он должен был остаться.»

Практика 15–45 13 / 22

Пять запросов в коллекции vibe-apis

  1. GET catfact.ninja/fact — самый простой JSON без авторизации. Возвращает случайный факт о кошках.
  2. GET openweathermap.org/data/2.5/weather с ?q=Moscow&appid={{openweather_key}} — ключ сохраняем в environment, не в URL.
  3. POST dummyjson.com/products/add с телом {"title": "AI-бот"} — учимся отправлять JSON в body.
  4. GET с неправильным ключом — ловим 401 Unauthorized. Разбираем, почему сервер так сказал.
  5. Query vs Path — разбор: /users?id=42 vs /users/42. Когда что.
Environments 14 / 22

Ключ API — это пароль

В коллекцию хардкодить нельзя — коллекция уйдёт в Git, а Git — публичный. Решение: environments.

❌ Так — нельзя

GET /weather?q=Moscow&appid=abc123realkey
http

Ключ улетает в git commit. Боты мониторят GitHub в реальном времени — украдут через минуты.

✓ Так — правильно

GET /weather?q=Moscow&appid={{openweather_key}}
http

Значение в файле environments/local.bru, который в .gitignore. В коллекции — только имя.

Правило на всю жизнь: если коммитнул ключ в публичный репо — сразу отзывай его в кабинете сервиса и выпускай новый. Боты находят ключи за минуты.

Сложности 17 / 22

Что обычно ломается

OpenWeather не отвечает

Ключ ещё не активирован

Активация занимает 10–60 минут. Не ждём — берём резервный DummyJSON, он без ключа.

Bruno не сохраняет в папку

Local Collection

При создании коллекции нужно выбрать Local Collection, а не облачную. Иначе коллекция живёт в памяти Bruno.

401 Unauthorized

Пробел в ключе

9 из 10 случаев — скопировал ключ вместе с пробелом в начале или в конце. Пересчитай символы — сравни с панелью сервиса.
