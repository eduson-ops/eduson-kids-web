Теория · 1 из 3

Как устроен стек

Редактор не «умный» сам по себе. Ум ему даёт плагин, а плагин говорит с движком, который держит модель.

Continue.dev шлёт HTTP на localhost:11434 — туда, где ждёт Ollama. Она держит модель в RAM и отвечает JSON-ом.

Теория · 2 из 3

Cloud-LLM vs Local-LLM

Правило дня: бытовые вопросы — в локальную. Серьёзные архитектурные — в облачную. Использовать обе, не одну.

Теория · 3 из 3

SSH-ключ: паспорт разработчика

GitHub не пустит тебя с паролем в 2026 — только с ключом. Одна половинка — у тебя, вторая — у GitHub.

Приватный ключ

Лежит на твоём ноуте в ~/.ssh/id_ed25519. Никогда не показываешь, не отправляешь, не коммитишь.

Публичный ключ

Файл id_ed25519.pub. Открыт всем. Это его ты копируешь в настройки GitHub.

Demo · живой запуск

Continue.dev просит Qwen написать FizzBuzz

Ctrl+L → откроется панель чата → пишешь промпт:

Markdown
## Промпт в Continue.dev Напиши на Python функцию fizzbuzz(n), которая печатает числа от 1 до n, но кратные 3 заменяет на "Fizz", кратные 5 на "Buzz", кратные 15 на "FizzBuzz".

Ответ прилетает из localhost:11434 — это твоя Ollama на том же ноуте.

Qwen 2.5 отвечает за ~3 секунды:

Python
def fizzbuzz(n: int) -> None: for i in range(1, n + 1): if i % 15 == 0: print("FizzBuzz") elif i % 3 == 0: print("Fizz") elif i % 5 == 0: print("Buzz") else: print(i) if __name__ == "__main__": fizzbuzz(20)

Проверка: выдерни провод интернета → попроси Continue ещё раз → всё равно работает. Вот что значит локальная модель.

Step 1 · VS Code + плагины

Ставим редактор и расширения

  1. Идём на code.visualstudio.com, качаем, ставим (Windows — .exe, Mac — .dmg).
  2. Запускаем. В левой панели иконка «квадратики» → Extensions.
  3. Ставим три расширения: Continue, Python, Pylance.
  4. Перезапускаем VS Code. Слева должна появиться иконка Continue.
Подсказка: что делать, если не видно расширений?
Открой меню View → Extensions или нажми Ctrl+Shift+X. В поле поиска сверху вбей название — VS Code найдёт в маркетплейсе. Кнопка Install справа от каждого.
Подсказка: чем Pylance отличается от Python?
Расширение Python даёт базовый запуск и отладку. Pylance — это language server: подсказки типов, автокомплит, быстрая навигация. Ставь оба.
Step 2 · Ollama + модель

Ставим движок и тащим модель

  1. Скачать инсталлер с ollama.com, установить.
  2. Открыть терминал, проверить: ollama --version.
  3. Запустить pull модели (~5 GB, 3–10 минут):
  4. Проверить: ollama list — должна быть в списке.
# проверяем установку
ollama --version
# ollama version 0.5.x

# тащим модель Qwen 2.5 7B (~5 GB)
ollama pull qwen2.5:7b

# проверяем
ollama list
# NAME           SIZE    MODIFIED
# qwen2.5:7b     4.7 GB  now

# быстрый тест из терминала
ollama run qwen2.5:7b "привет!"
bash
Подсказка: «Ollama виснет при установке модели»
1) Проверь свободное место — нужно минимум 5 GB. 2) Закрой Chrome и другие пожиратели RAM. 3) Ставь с проводного интернета, Wi-Fi рвётся на больших файлах.
Подсказка: «Порт 11434 занят»
Windows: netstat -ano | findstr 11434 → увидишь PID → taskkill /PID <pid> /F. Mac/Linux: lsof -i :11434 → kill -9 <pid>.
Step 3 · связываем Continue с Ollama

Конфиг Continue.dev

Открываешь иконку Continue в VS Code → шестерёнка → config.json. Вставляешь:

{
  "models": [
    {
      "title": "Qwen 2.5 7B (local)",
      "provider": "ollama",
      "model": "qwen2.5:7b",
      "apiBase": "http://localhost:11434"
    }
  ],
  "tabAutocompleteModel": {
    "title": "Qwen 2.5 autocomplete",
    "provider": "ollama",
    "model": "qwen2.5:7b"
  }
}
json

Сохраняешь файл. Открываешь чат Continue (Ctrl+L), пишешь любой вопрос — ответ прилетит из локальной модели. Проверка: отключи интернет → чат по-прежнему работает.

Step 4 · SSH-ключ и GitHub

Генерим ключ и скармливаем GitHub

Команды

# генерим пару ключей
ssh-keygen -t ed25519 -C "your@email.com"
# жмём Enter на все вопросы

# смотрим публичную часть (её копируем в GitHub)
cat ~/.ssh/id_ed25519.pub
# ssh-ed25519 AAAAC3Nza... your@email.com

# проверяем, что GitHub принял ключ
ssh -T git@github.com
# Hi username! You've successfully
# authenticated...
bash

В браузере (через VPN)

  1. Заходим на github.com/settings/keys.
  2. Кнопка New SSH key.
  3. Title — my-laptop. Key — вставляем всё, что вывел cat .pub.
  4. Жмём Add SSH key.
  5. Тестируем ssh -T git@github.com.
Подсказка: «SSH-ключ не принимается GitHub»
Скорее всего скопировал без префикса ssh-ed25519 или с лишними пробелами/переносами. Делай cat ~/.ssh/id_ed25519.pub ещё раз и копируй ВСЁ в одну строку, включая email в конце.
Подсказка: «VPN не тянет GitHub»
Имей заранее второй VPN-профиль. Если основной упал — переключаешься без паники. GitHub без VPN не открывается — это нормально, работаем через школьный.

Troubleshooting · если что-то пошло не так

Три главных затыка

Ollama

Зависает pull модели

Нет места (нужно 5 GB), или убивает слабый Wi-Fi. Закрываешь Chrome, переключаешься на провод, запускаешь заново.

SSH

GitHub не принимает ключ

Копировал не весь .pub-файл или с лишними пробелами. Повтори cat ~/.ssh/id_ed25519.pub и копируй всю строку.

VPN

GitHub не грузится

Основной VPN упал — держи в запасе второй профиль. GitHub без VPN сейчас не открывается — это норма.
