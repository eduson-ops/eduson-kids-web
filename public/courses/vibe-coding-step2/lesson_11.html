09 / 22url = бейдж
Как это выглядит

Превью на GitHub

vibe-api

python3.11 licenseMIT uptime99.8% FastAPI0.115

REST-обёртка над GigaChat и YandexGPT. Переводит, саммаризует, генерирует гороскопы. Живёт на my-vibe.ru 24/7 через systemd + Nginx + Let's Encrypt.

→ Демо: https://my-vibe.ru/docs

Фичи

— POST /translate — перевод через YandexGPT, 300 мс
— POST /summarize — саммари через GigaChat
— GET /horoscope?sign=lev — гороскоп по знаку
— rate-limit 10 req/min, логи в journalctl

Запуск локально

git clone · cd api · python -m venv .venv · pip install -r requirements.txt · cp .env.example .env · uvicorn main:app --reload

Как запустить локально

Золотой блок quick start

README.md · ## Запуск локально
git clone git@github.com:vibe-kid/vibe-step2-ivan.git
cd vibe-step2-ivan/api

python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate

pip install -r requirements.txt
cp .env.example .env          # вставь свои ключи GigaChat и Yandex
uvicorn main:app --reload

# Открой http://localhost:8000/docs
bash

Проверь сам

Скопируй команды. Запусти на чистой машине. Всё должно заработать без твоих объяснений.

.env.example обязателен

Файл с пустыми ключами. Чтобы человек понимал, что нужно заполнить. Сам .env — в .gitignore.

requirements.txt — свежий

pip freeze > requirements.txt перед пушем. Старые версии — причина 80% «у меня не работает».

  "openapi": "3.1.0",
  "info": {
    "title": "vibe-api",
    "version": "0.1.0"
  },
  "paths": {
    "/horoscope": {
      "get": {
        "summary": "Получить гороскоп по знаку",
        "parameters": [
          { "name": "sign", "in": "query", "required": true,
            "schema": { "type": "string" } },
          { "name": "source", "in": "query",
            "schema": { "type": "string", "enum": ["gigachat", "yandex"] } }
        ],
        "responses": {
          "200": { "description": "Successful Response" },
          "400": { "description": "Unknown sign" },
          "429": { "description": "Rate limit exceeded" }
        }
      }
    }
  }
}

Сложности и фиксы

Что пойдёт не так

Бейдж не грузится

URL должен быть HTTPS. Иногда GitHub кэширует старую версию — подожди минуту и обнови страницу.

Скрин читается плохо

Обрежь до релевантной области. Ширина до 1600 px. Формат PNG или WebP.

README слишком длинный

Правило: до 300 строк. Больше — выноси в docs/ или GitHub Wiki.

Команды не копируются

Обернуть в ```bash-блок. На GitHub появится кнопка «copy» автоматически.

Скрин не отображается

Проверь путь: относительный ./docs/swagger.png. Абсолютный с твоего домена — рискованно.

Uptime-бейдж красный

Сайт реально упал. Сначала фикси сервис, потом добавляй бейдж. Иначе честно-показываешь, что не живой.
