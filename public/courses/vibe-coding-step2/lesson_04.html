Теория · 10 минут

OAuth 2.0 — пропуск в здание

1. Ты показываешь паспорт

Отправляешь client_id и client_secret в /oauth. Это Basic Auth.

2. Тебе дают бейдж

В ответ — access_token. Это Bearer-токен, который живёт 30 минут.

3. Ходишь по этажам

В каждый запрос кладёшь заголовок Authorization: Bearer <token>. GigaChat пускает.

# Шаг 1: получить токен
curl -X POST "https://ngw.devices.sberbank.ru:9443/api/v2/oauth" \
  -H "Authorization: Basic $BASE64_ID_SECRET" \
  -H "RqUID: $UUID_V4" \
  -d "scope=GIGACHAT_API_PERS"

# Ответ:
# { "access_token": "eyJhbGc...", "expires_at": 1745193600000 }

# Шаг 2: идём с токеном в /chat/completions
curl -X POST "https://gigachat.devices.sberbank.ru/api/v1/chat/completions" \
  -H "Authorization: Bearer eyJhbGc..." \
  -H "Content-Type: application/json" \
  -d '{"model":"GigaChat","messages":[...]}'

!

Почему токен, а не пароль напрямую?

Украдут токен — потеряешь максимум 30 минут. Украдут client_secret — пиши в Сбер, восстанавливай неделю. Поэтому Сбер разработал для нас это разделение.

Паттерн: кэш токена

Токен живёт 30 минут — кэшируем

Python
import time import httpx import uuid import base64 _token_cache = {"value": None, "expires_at": 0} async def get_gigachat_token() -> str: now = time.time() # Если токен ещё жив — переиспользуем if _token_cache["value"] and _token_cache["expires_at"] > now + 60: return _token_cache["value"] # Иначе — запрашиваем новый auth = base64.b64encode( f"{GIGACHAT_CLIENT_ID}:{GIGACHAT_CLIENT_SECRET}".encode() ).decode() async with httpx.AsyncClient(verify=False) as client: r = await client.post( "https://ngw.devices.sberbank.ru:9443/api/v2/oauth", headers={ "Authorization": f"Basic {auth}", "RqUID": str(uuid.uuid4()), "Content-Type": "application/x-www-form-urlencoded", }, data={"scope": "GIGACHAT_API_PERS"}, ) r.raise_for_status() data = r.json() _token_cache["value"] = data["access_token"] # Кэшируем на 25 минут (токен живёт 30, запас 5) _token_cache["expires_at"] = now + 25 * 60 return _token_cache["value"]

/translate — переводчик на YandexGPT

Python
from pydantic import BaseModel from fastapi import HTTPException class TranslateRequest(BaseModel): text: str target: str # "en", "fr", "de", ... @app.post("/translate") async def translate(req: TranslateRequest): system_prompt = f"Ты профессиональный переводчик. Переведи текст на язык: {req.target}. Верни ТОЛЬКО перевод, без пояснений." payload = { "modelUri": f"gpt://{YANDEX_FOLDER_ID}/yandexgpt-lite/latest", "completionOptions": {"temperature": 0.2, "maxTokens": 1000}, "messages": [ {"role": "system", "text": system_prompt}, {"role": "user", "text": req.text}, ], } async with httpx.AsyncClient(timeout=30.0) as client: r = await client.post( "https://llm.api.cloud.yandex.net/foundationModels/v1/completion", headers={"Authorization": f"Api-Key {YANDEX_API_KEY}"}, json=payload, ) if r.status_code == 429: raise HTTPException(503, "Переводчик занят, попробуй через минуту") r.raise_for_status() answer = r.json()["result"]["alternatives"][0]["message"]["text"] return {"translated": answer}

class SummarizeRequest(BaseModel): url_or_text: str @app.post("/summarize") async def summarize(req: SummarizeRequest): token = await get_gigachat_token() # кэш внутри payload = { "model": "GigaChat", "temperature": 0.3, "messages": [ {"role": "system", "content": "Ты — саммаризатор. Сделай краткое саммари в РОВНО 3 предложениях. Рычаг управления

Температура — регулятор креатива

Один параметр, который меняет характер ответа полностью. Запомни таблицу — сэкономишь часы отладки промптов.

ЗадачаТемператураПример
Перевод, факты0.0 – 0.2Точность важнее вариативности
Саммари, резюме0.2 – 0.4Чуть «оживить», но не выдумать

T

Правило большого пальца

Чем объективнее задача — тем ниже температура. Чем субъективнее — тем выше. Не нужно знать теорию вероятностей — достаточно помнить это.

Надёжность

Четыре сценария «что может пойти не так»

401

Токен протух

GigaChat вернул 401. Сбрасываем кэш, следующий вызов регенерит токен. Пользователю — 503 и просьба повторить.

429

Rate limit

LLM исчерпала лимит. Возвращаем 503 с сообщением «сервис занят, попробуй через минуту». Не пытайся retry в цикле.

⏱

Таймаут

LLM думает больше 60 секунд. httpx.AsyncClient(timeout=60.0) — и ловим httpx.ReadTimeout. Пользователю — 503.

∅

Пустой ответ

LLM вернула content: "". Проверяем длину, если пусто — fallback-сообщение «не удалось сгенерировать ответ».

Обработка ошибок — это не «лишнее», это то, что отличает прод от студенческой поделки.

В /translate добавь параметр tone: Literal["official","casual"]. Если official — системный промпт «переведи деловым языком». Если casual — «переведи разговорным стилем, можно со сленгом».

Бонус · для опережающих

/compare — обе модели параллельно

Зовём GigaChat и YandexGPT одновременно через asyncio.gather. Возвращаем оба ответа рядом — пользователь выбирает, какой лучше.

Python
import asyncio async def call_gigachat(text: str) -> str: token = await get_gigachat_token() # ... запрос к GigaChat return giga_answer async def call_yandexgpt(text: str) -> str: # ... запрос к YandexGPT return yandex_answer @app.post("/compare") async def compare(req: SummarizeRequest): # Параллельный запуск, ждём оба ответа одновременно giga, yandex = await asyncio.gather( call_gigachat(req.url_or_text), call_yandexgpt(req.url_or_text), return_exceptions=True, # если одна упадёт — не тащим за собой вторую ) return { "gigachat": str(giga), "yandexgpt": str(yandex), }
