Программист не копирует одно и то же 100 раз. Программист пишет цикл — и робот-помощник делает это за него. Сегодня научимся.

Проблема

01Зачем вообще цикл?

Представь: надо вывести «Привет» 100 раз. Копировать print() сто раз? Серьёзно?

Без цикла — боль
Python
print("Привет") print("Привет") print("Привет") # ... и так 97 раз 😭 print("Привет")
С циклом — элегантно
Python
for i in range(100): print("Привет")
Правило: одно и то же повторяется? Значит, нужен цикл.

03Анатомия for

разбираем по частям
Python
for i in range(5): print(i)

for

Ключевое слово. «Начинаем цикл».

i

Переменная-коробочка. В неё Python по очереди кладёт значения.

in range(5)

Последовательность: откуда брать значения.

отступ

4 пробела = «это тело цикла, выполняй на каждом шаге».

range()

04Три формы range()

1 аргумент
Python
range(5) # 0, 1, 2, 3, 4

От 0 до 5, не включая 5.

2 аргумента
Python
range(1, 10) # 1, 2, 3, ..., 9

От 1 до 10, не включая 10.

3 аргумента
Python
range(2, 21, 2) # 2, 4, 6, ..., 20

С шагом 2. Так ловим чётные.

⚠️ Правило десятки: range(1, 10) — десятки НЕТ. Хочешь десятку — пиши range(1, 11).
Пошагово

05Что делает Python на каждом шаге

код
Python
for i in range(4): print("шаг", i)

Python берёт значения из range(4) по очереди и кладёт в i. На каждом шаге выполняет тело цикла.

что происходит
i = 0→шаг 0
i = 1→шаг 1
i = 2→шаг 2
i = 3→шаг 3
стоп→4 не входит
Не только числа

06for по списку

for перебирает не только числа. Он умеет ходить по любой последовательности: список, строка, кортеж.

код
Python
friends = ["Аня", "Боря", "Витя"] for name in friends: print("Привет,", name + "!")
вывод
Python
Привет, Аня! Привет, Боря! Привет, Витя!
💡 Имя переменной — по смыслу. Не i, а name.
Продвинутый приём

07Номер + элемент = enumerate()

с enumerate — красиво
Python
cities = ["Москва", "Казань", "Уфа", "Сочи"] for idx, city in enumerate(cities, start=1): print(idx, city)
без enumerate — по-старому
Python
cities = ["Москва", "Казань", "Уфа", "Сочи"] i = 1 for city in cities: print(i, city) i += 1
Оба способа работают. enumerate — короче. Но главное — понять, что за ним стоит.

10Города с номерами

Возьми список городов и выведи каждый с его номером.

способ А — enumerate
Python
cities = ["Москва", "Казань", "Уфа", "Сочи"] for idx, city in enumerate(cities, 1): print(idx, city)
способ Б — счётчик
Python
cities = ["Москва", "Казань", "Уфа", "Сочи"] n = 1 for city in cities: print(n, city) n += 1

Вывод одинаковый: 1 Москва · 2 Казань · 3 Уфа · 4 Сочи

11Сумма от 1 до 100

Паттерн «аккумулятор»: переменная-копилка, в которую каждый шаг добавляется новое значение.

код
Python
total = 0 for n in range(1, 101): total += n print(total)
вывод
Python
5050
🎓 Задачу в уме решил Карл Гаусс в 7 лет. Ты — в несколько строк кода.

12Таблица умножения

код

n = int(input("Введи число: ")) print(f"Таблица умножения на {n}:") for i in range(1, 11): print(n, "×", i, "=", n * i)
пример запуска
Python
Введи число: 7 Таблица умножения на 7: 7 × 1 = 7 7 × 2 = 14 7 × 3 = 21 ... 7 × 10 = 70

1410 фраз, которые работают

01Цикл — это твой робот-помощник: сказал 10 раз — сделает 10 раз.
02range(5) — это [0,1,2,3,4], а не [1..5]. Python стартует с нуля.
03Переменная i — это коробочка, которая обновляется сама.
04Отступ — это вход в тело цикла. Забыл отступ — Python обидится.
05Называй переменную по смыслу: i, n, city, name.
06Сначала напиши на листе, что должен вывести цикл на первых 3 шагах.
07Хочешь десятку в диапазоне — пиши range(1, 11), не range(1, 10).
08for перебирает любую последовательность: список, строку, кортеж.
09Если зациклило — проверь: переменная в условии точно меняется?
10Правило 5 минут: подумай сам, прежде чем спрашивать Алису.
Траблшутинг

15Что пойдёт не так (и как чинить)

⚠️ Путают range(5) и range(1, 5).
Рисуй на листе: «диапазон» vs «отрезок». Покажи оба визуально.
⚠️ Забыли отступ → IndentationError.
Читаем traceback вместе. Python сам говорит, в какой строке.
⚠️ Пишут for i = 0 to 5 — привычка из Scratch.
«Python говорит по-своему». Без осуждения — спокойная поправка.
⚠️ Не понимают n ** 2 — что за две звезды?
Повтор степени из урока 2: ** = «в степени».
⚠️ print(i) написали вне цикла — вывод один раз вместо пяти.
Спроси: «В какой момент выполняется этот print?» Пусть сам найдёт.
⚠️ Забыли двоеточие после range(...).
Правило: после for ... in ... всегда «:», как вход в комнату.

21Три вещи, которые ты теперь знаешь

🔁

Что такое for

Цикл, когда число повторений известно заранее. Робот-помощник для повторов.

🎯

Что делает range()

Создаёт последовательность чисел. Помнит: верхняя граница не входит.

➡️

Зачем отступы

Отступ — это вход в тело цикла. Без отступа Python не понимает, что повторять.
