Карта состояний

Четыре комнаты одной игры

🏠
menu
Заставка. Ждём пробел. Игрок смотрит и думает.
🎮
playing
Яблоки падают, корзинка ловит, счёт растёт.
⏸️
paused
Игра замерла. Логика спит, картинка есть.
💀
gameover
Жизни кончились. Счёт и кнопка «заново».
menu — SPACE → playing — P ⇄ — paused — lives=0 → gameover — SPACE → menu

Одна и та же функция draw() рисует 4 разные картинки — в зависимости от значения game_state.

Клавиатура · первое знакомство

on_key_down — пульт игры

SPACE
Из меню → начать игру. Из gameover → рестарт.
P
Из playing → пауза. Из paused → продолжить.
ESC
Выйти из игры (автоматически PGZ).
⚡
Событие, а не состояние
on_key_down сработает РОВНО один раз за нажатие. Не 60 раз в секунду, как keyboard.space.
on_key_down(key)
Python
def on_key_down(key): global game_state, score, lives if key == keys.SPACE: if game_state == 'menu': game_state = 'playing' elif game_state == 'gameover': reset_game() # обнуляем и в меню elif key == keys.P: if game_state == 'playing': game_state = 'paused' elif game_state == 'paused': game_state = 'playing'
update() с охраной

Логика спит, когда не её время

def update(dt)
Python
def update(dt): global score, lives, game_state # ГВАРД: в меню / паузе / gameover — ничего не делаем if game_state != 'playing': return for apple in apples[:]: apple.y += apple.speed # поймали? if basket.colliderect(apple): apples.remove(apple) score += 1 # улетело за экран? elif apple.y > HEIGHT: apples.remove(apple) lives -= 1 if lives <= 0: game_state = 'gameover'

Одна строка return наверху — и меню больше не превращается в мгновенный gameover.

Новый инструмент: текст

screen.draw.text — надписи в игре

примеры текста
Python
# Счёт в углу screen.draw.text( f'Score: {score}', (10, 10), color='white', fontsize=36 ) # Заголовок по центру screen.draw.text( 'ПОЙМАЙ ЯБЛОКО', center=(400, 200), fontsize=60, color='yellow' ) # Подсказка screen.draw.text( 'Нажми ПРОБЕЛ', center=(400, 320), fontsize=28, color=(124, 58, 237) # RGB тоже можно )

3 обязательных аргумента

  1. Что писать — строка или f'...'
  2. Куда — (x, y) угол или center=(x, y)
  3. Как — color, fontsize

🎨 Цвета, которые работают

Строкой: 'white', 'red', 'yellow', 'cyan', 'lime'.
Кортежем RGB: (255, 200, 0). Проверяй контраст с фоном!

Рестарт без перезапуска

Функция reset_game() — новый старт

def reset_game()
Python
def reset_game(): global game_state, score, lives game_state = 'playing' score = 0 lives = 3 apples.clear() # ← ВАЖНО! иначе яблоки с прошлой игры останутся
⚠️
Типичный баг: «яблоки остались от прошлой игры»
Забыли apples.clear(). Разбери это с ребёнком через print(len(apples)) — видно, сколько «призраков» осталось.

Пауза на P

draw() для paused
Python
elif game_state == 'paused': # сначала рисуем игру как обычно screen.blit('background', (0, 0)) basket.draw() for apple in apples: apple.draw() # поверх — полупрозрачный прямоугольник screen.draw.filled_rect( Rect(0, 0, WIDTH, HEIGHT), (0, 0, 0, 150) # полупрозрачный чёрный ) # крупная надпись screen.draw.text( '⏸ ПАУЗА', center=(WIDTH/2, HEIGHT/2), fontsize=80, color='white' )

🔑 Две идеи паузы

  • update() уже защищён гвардом — логика сама остановится
  • draw() продолжает рисовать, просто другую картинку поверх

💬 Что сказать ребёнку

«Смотри: update заснул, а draw продолжает работать. Игра как фотография — видно, но ничего не двигается».

Сложности и решения

4 грабли, на которые наступают все

🧟

Яблоки остались от прошлой игры

Забыли apples.clear() в reset_game(). Проверка: print(len(apples)) в меню — должно быть 0.

🙈

Счёт не растёт

score += 1 внутри цикла, но после apples.remove, строка недостижима. Переставляем местами.

👻

Текст не виден

color='white' на светлом фоне. Меняем на 'black' или 'yellow'. Проверяем контраст.

🧲

Состояние не меняется

Забыт global game_state внутри функции. Python создал локальную переменную с тем же именем.
