Что это

Словарь — это пара ключ → значение

Открой толковый словарь. Слева слово — ключ. Справа объяснение — значение.

В Python то же самое: записываем в фигурных скобках {}, разделяем двоеточием :, пары — через запятую.

Шаг 1 · Создание

Первый словарь за одну строку

Фигурные скобки. Пары через двоеточие. Пары между собой — через запятую.

Ключ обычно — строка. Значение — что угодно: число, строка, список, другой словарь.

{} ключ:значение через запятую
hero.py · карточка героя
Python
hero = { "name": "Маг", "hp": 100, "mp": 80, "урон": 25, } print(hero["name"]) # Маг print(hero["hp"]) # 100
Правила

Ключи и значения: что можно

🔑 Ключи

  • Уникальные — дубликат затрёт значение
  • Обычно строка, реже — число
  • Список ключом быть не может
  • Регистр важен: "HP" ≠ "hp"

💎 Значения

  • Могут повторяться сколько угодно
  • Любого типа: число, строка, список
  • Можно класть целый другой словарь
  • Менять сколько угодно
⚠️
Дубликат затирает
{"a": 1, "a": 2} превратится в {"a": 2}. Python молча оставит только последнее — это частая ошибка.
Шаг 2 · Чтение

Три способа достать значение

d["key"]

Короткий доступ. Но если ключа нет — KeyError и программа падает.

d.get("key")

Если ключа нет — вернёт None и программа не упадёт.

d.get("key", "нет")

Если ключа нет — вернёт ваш запасной ответ. Самый безопасный способ.

dictionary.py · справочник RU → EN
Python
d = {"кот": "cat", "пёс": "dog", "мышь": "mouse"} print(d["кот"]) # cat print(d.get("кот")) # cat print(d.get("тигр")) # None (не упадёт!) print(d.get("тигр", "нет в словаре")) # нет в словаре
Шаг 3 · Изменение

Добавить · Изменить · Удалить

friends.py · словарь меняется на ходу
Python
friends = {"Петя": 10, "Аня": 11} # ➕ добавить пару friends["Миша"] = 12 # ✏️ изменить значение (ключ уже есть) friends["Петя"] = 11 # ➖ удалить пару del friends["Аня"] # 🎯 или pop — удалит и вернёт значение age = friends.pop("Миша") print(friends) # {'Петя': 11}
Методы

Три главных метода: .keys · .values · .items

.keys()

Все ключи словаря

['Петя', 'Аня']

.values()

Все значения словаря

[10, 11]

.items()

Все пары сразу

[('Петя',10), ('Аня',11)]

loop.py · перебор словаря циклом
Python
friends = {"Петя": 10, "Аня": 11, "Миша": 12} for name, age in friends.items(): print(f"{name} — {age} лет") # Петя — 10 лет # Аня — 11 лет # Миша — 12 лет
🧠
Ты будешь писать это миллион раз
for key, value in d.items(): — запомни наизусть. Это рабочая лошадка Python-разработчика.

Уровень выше

Словарь внутри словаря

Значением можно положить целый другой словарь. Получается карточка героя со всеми параметрами.

Доступ — двойные скобки: сначала герой, потом параметр.

вложенность heroes["Маг"] heroes["Маг"]["hp"]
heroes.py · карточки персонажей
Python
heroes = { "Маг": {"hp": 100, "mp": 80, "урон": 25}, "Воин": {"hp": 150, "mp": 20, "урон": 35}, "Лучник":{"hp": 120, "mp": 40, "урон": 30}, } print(heroes["Маг"]["hp"]) # 100 print(heroes["Воин"]["урон"]) # 35 for name, stats in heroes.items(): print(name, "→", stats["hp"], "hp")
Когда что брать

Список или словарь?

📋

Список

Когда важен порядок и всё однотипное.

marks = [5, 4, 5, 3, 4] — оценки за четверть

Доступ по индексу: marks[0] → 5

📖

Словарь

Когда нужен быстрый поиск по имени.

contacts = {"мама": "+7...", "брат": "+7..."}

Доступ по ключу: contacts["мама"]

🚫
Частая ошибка: d[0] у словаря
Словарь НЕ индексируется числом (если ключ не число). Только по ключу. Индексов тут нет.
Сложности

5 частых ошибок

1. d[0] — индекса нет

Словарь ищет только по ключу. Если ключ не 0 — упадёт KeyError.

2. Забыл двоеточие

{"name" "Маг"} → SyntaxError. Между ключом и значением — :.

3. Одинаковые ключи

Второе значение молча затрёт первое. Данные потеряны.

4. Путаница keys ↔ values

Мнемоника: keys сверху (слова), values снизу (определения).

5. for x in d даёт ключи, не пары

Если нужны пары — используй for k, v in d.items():. Это ключевая идиома языка.

Парсинг расписания

Ключ — день недели, значение — список уроков.

Пользователь вводит день — программа печатает уроки по порядку.

словарь + список for + enumerate
schedule.py
Python
schedule = { "пн": ["матем", "русский", "ИЗО"], "вт": ["физра", "англ", "история"], "ср": ["матем", "литература"], } day = input("День недели: ").lower() lessons = schedule.get(day, []) if not lessons: print("Выходной или день не найден") else: for i, lesson in enumerate(lessons, 1): print(f"{i}. {lesson}")

Три правила, которые ты унесёшь

🔑

1. Ключ → Значение

Словарь — это пары. Ключ уникальный, значение — любое.

✏️

2. {} и :

Фигурные скобки. Пары через двоеточие. Пары между собой — через запятую.

🛡️

3. .get() безопаснее []

Если не уверен, что ключ есть — .get() не уронит программу.
