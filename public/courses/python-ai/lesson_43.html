Как оптимизировать UX игры

Не про код — про заботу об игроке. Кнопка вместо текста, ховер, подсказки, переключатель звука. Игра перестаёт быть прототипом — становится игрой.

Знакомство с понятием

UX — это забота об игроке

UX = User Experience = «опыт пользователя».

Проще: игрок не должен гадать, что делать. Всё очевидно с первого взгляда.

«Игрок не умеет читать код. Если ему непонятно за первые 5 секунд — он закроет игру». — правило UX
👁

Три железных правила UX

01

Всегда понятно, что делать

Игрок за 5 секунд должен сам догадаться: куда нажать, куда двигаться, зачем всё это.

Можно выйти и выключить

⚡
UX-принцип 100 мс
После каждого действия игрока — видимая реакция за 100 миллисекунд. Иначе мозг думает «кнопка сломана».
UX-принцип #2

Почему 100 миллисекунд решают всё

Мозг игрока ждёт реакцию меньше 1/10 секунды.

Если реакция пришла позже — сомнение: «сломано? я промахнулся?».

В коде это значит: клик → сразу звук И смена картинки И счёт +1. Все вместе, без пауз.

Шаг 1 · Совместный кодинг 15–30

Кнопка — это просто Actor

Никакой магии. Тот же Actor, что и корзинка с яблоком.

  • Создаём переменную
    В начале файла рядом с basket.
  • Рисуем в меню
    Внутри if game_state == 'menu':.
  • Текст-подпись сверху
    Название игры остаётся — кнопка только для старта.
game.py · добавляем кнопку 
# рядом с basket = Actor(...)
play_btn = Actor('btn_play')
play_btn.pos = (400, 300)

def draw():
    if game_state == 'menu':
        screen.blit('menu_bg', (0, 0))
        screen.draw.text('Поймай яблоко',
                         center=(400, 180),
                         fontsize=56, color='white')
        play_btn.draw()
    elif game_state == 'playing':
        # ... старый код игры
        pass
Шаг 2 · Новый метод

Как понять, попал ли клик в кнопку?

actor.collidepoint(pos) — True, если точка внутри картинки.

🎯 Мышь — это точка

PyGame Zero передаёт pos = (x, y) в on_mouse_down. Это и есть точка клика.

«collidepoint — это как ткнуть пальцем в карту. Попал в страну — True, в океан — False». — аналогия на уроке
game.py · обработка клика 
def on_mouse_down(pos, button):
    global game_state

    # Только в меню слушаем кнопку старта
    if game_state == 'menu':
        if play_btn.collidepoint(pos):
            game_state = 'playing'
            sounds.click.play()

    # В игре клики пусть делают своё
    elif game_state == 'playing':
        pass

Типичная ошибка
Забыли else — кнопка остаётся «горящей» после того как курсор ушёл. Две ветки, не одна!
game.py · on_mouse_move 
def on_mouse_move(pos):
    # Корзинка — как раньше, следит за мышью
    if game_state == 'playing':
        basket.x = pos[0]

    # Ховер кнопки — только в меню
    if game_state == 'menu':
        if play_btn.collidepoint(pos):
            play_btn.image = 'btn_play_hover'
        else:
            play_btn.image = 'btn_play'
Шаг 4 · Подсказка первые 3 секунды

Онбординг — помогаем в старте

Подсказка нужна только в первые 3 секунды. Потом мешает.

Score: 0
🔊
💡 Двигай мышкой — лови яблоки!
🚫
Не вызывай clock.schedule в draw()
Он сработает 60 раз в секунду. Таймеры накопятся. Делаем один раз — в момент старта игры.
game.py · подсказка с таймером 
show_hint = False

def hide_hint():
    global show_hint
    show_hint = False

def on_mouse_down(pos, button):
    global game_state, show_hint
    if game_state == 'menu' and play_btn.collidepoint(pos):
        game_state = 'playing'
        show_hint = True
        clock.schedule(hide_hint, 3.0)  # один раз!

def draw():
    # ... игра
    if game_state == 'playing' and show_hint:
        screen.draw.text('Двигай мышкой — лови яблоки!',
                         center=(400, 80),
                         fontsize=28, color='white')
Шаг 5 · Звук вкл/выкл

Иконка в углу — свобода игрока

Принцип #3 в действии: игрок может выключить.

🔊

sound_on = True

Музыка играет, звуки срабатывают.

🔇

sound_on = False

Полная тишина — и музыка, и эффекты.

«Без кнопки звук вкл/выкл — меня забанят родители». — фраза преподавателя
game.py · переключатель 
sound_btn = Actor('icon_sound_on')
sound_btn.pos = (760, 40)
sound_on = True

def on_mouse_down(pos, button):
    global sound_on
    if sound_btn.collidepoint(pos):
        sound_on = not sound_on
        if sound_on:
            sound_btn.image = 'icon_sound_on'
            music.play('bg')
        else:
            sound_btn.image = 'icon_sound_off'
            music.stop()

def draw():
    # ... игра
    sound_btn.draw()   # рисуется всегда поверх
Ключевые фразы

Что говорим детям дословно

«UX — это забота об игроке. Игрок не умеет читать код». главная формула урока
«Если игроку непонятно, что делать в первые 5 секунд — игра проиграна». правило 5 секунд
«Кнопка — это картинка, реагирующая на курсор». определение кнопки
«Ховер — маленький фидбек, говорящий „меня можно нажать"». ховер = приглашение
«Подсказка — только вначале. Потом мешает». правило подсказок
«Звук вкл/выкл — обязательная кнопка. Иначе меня забанят родители». и это правда
Типичные баги и как их чинить

Три бага, которые всегда случаются

❌

Клик не попадает

Симптом: нажимаю на кнопку — ничего.

Причина: позиция Actor не совпадает с картинкой.

Лечение: print(pos, play_btn.pos) и сверить.

⚠

Ховер мигает

Симптом: кнопка «горит» даже когда курсор ушёл.

Причина: забыли else — вернуть обычную картинку.

Лечение: две ветки if / else, без вариантов.

🔁

Подсказка не исчезает

Симптом: висит всю игру.

Причина: clock.schedule вызван в draw() — 60 таймеров в секунду.

Лечение: перенести в on_mouse_down при старте.

Звёздная задача · бонус

Fade-in — плавное появление меню

Вместо «бац — меню», делаем плавное проявление за 1 секунду.

  • Переменная fade_alpha от 255 к 0.
  • В update() уменьшаем на 5 каждый кадр.
  • В draw() рисуем чёрный прямоугольник с этой прозрачностью поверх всего.
✨
Бонус-эффект
Та же техника — для экранов gameover и pause. Полировка на уровне Steam-игр.
game.py · fade-in 
fade_alpha = 255  # полностью тёмный

def update():
    global fade_alpha
    if fade_alpha > 0:
        fade_alpha -= 5

def draw():
    # ... сначала всё остальное
    if fade_alpha > 0:
        screen.draw.filled_rect(
            Rect(0, 0, WIDTH, HEIGHT),
            (0, 0, 0, fade_alpha)
        )
