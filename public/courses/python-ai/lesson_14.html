Цикл while
— «делай, пока…»

Повторение без счётчика. Условие вместо количества. И самый опасный баг мира — бесконечный цикл. Остановить можно одной комбинацией клавиш.

Python 3.11+
GigaIDE Desktop
Ctrl+C — спасение
Метафора дня

Что такое while?

🛗 Лифт едет, пока не приедет на нужный этаж.

Он не считает «1-2-3 этаж». Он просто проверяет: «я уже там? нет? — едем дальше».

Это и есть while: «пока условие истинно — повторяй».

🎯

Когда выбираем while

  • Не знаю, сколько раз повторить
  • Жду события: «пока не введут стоп»
  • Условие меняется по ходу
  • «Работаем, пока есть деньги»

Сравнение

for vs while

🏃 for

  • 🎯 Знаю число повторений
  • 🔢 «Пробежать 10 кругов»
  • 📋 Иду по списку/диапазону
  • 🚂 Поезд по расписанию
  • for i in range(10):
VS

🌀 while

  • ❓ Не знаю, сколько раз
  • 🏋️ «Бегу, пока не устану»
  • ⏳ Жду события или условия
  • 🛗 Лифт едет до этажа
  • while n <= 5:

💡 Если знаешь сколько раз — бери for. Если знаешь до каких пор — бери while.

Синтаксис

Как устроен while

  • Ключевое слово while
    Начинается строка с while.
  • Условие
    Что-то, что может быть True или False.
  • Двоеточие :
    В конце строки. Без него — SyntaxError.
  • Отступ — 4 пробела
    Тело цикла. То, что повторится.
syntax.py
Python
n = 1 while n <= 5: # условие + двоеточие print(n) # тело с отступом n += 1 # двигаем счётчик! print("Готово") # это уже вне цикла
🐍
Python-стиль
Скобки вокруг условия не нужны: пишем while n <= 5:, а не while (n <= 5):. Второй вариант не ошибка, но так не пишут.

Главный трюк

Счётчик — ты сам его двигаешь

🔼 n += 1 — «увеличь на 1»

Это сокращённая запись. n += 1 — то же самое, что n = n + 1.

Python
n += 1 # n = n + 1 n -= 1 # n = n - 1 n *= 2 # n = n * 2

😱 Забыл двинуть — цикл вечен

Если не обновить переменную внутри цикла, условие никогда не станет ложным. Python будет крутиться, пока не нажмёшь Ctrl+C или не закроешь ноут.

Частая путаница

if или while?

🙋 if — вопрос один раз

«У тебя есть деньги?» — спросил кассир один раз. Ответил — дальше действие. Всё.

Python
money = 100 if money > 0: print("Покупаю 1 раз")

🔁 while — вопрос перед каждым шагом

«Пока есть деньги — покупай». Проверка перед каждой покупкой, снова и снова.

Python
money = 100 while money > 0: print("Покупаю ещё раз") money -= 30

📌 if выполнится 0 или 1 раз. while выполнится 0, 1 или много раз.

Практика 1 · 10 минут

🎯 Угадай число (без random)

guess.py
Python
secret = 7 guess = 0 # стартовое, любое != 7 while guess != secret: guess = int(input("Угадай число от 1 до 10: ")) if guess < secret: print("📉 Моё число больше") elif guess > secret: print("📈 Моё число меньше") print("🎉 Угадал! Это 7.")

🧠 Идея

«while guess != 7:» читаем как «пока не угадал, продолжай спрашивать».

⚠️ Что обсудить

Начальное значение guess = 0 — любое, лишь бы не равное secret. Иначе цикл даже не начнётся.

Практика 2 · 10 минут

Счётчик до «стоп»

Считаем слова, пока пользователь не напишет стоп.

while True: — «крутись всегда». break — «ломай цикл, вылезай». Подробнее — на уроке 15.

🚪
Без break — зависание
while True без break внутри — это гарантированный бесконечный цикл.
counter.py
Python
count = 0 while True: cmd = input("Слово (или 'стоп'): ") if cmd == "стоп": break # вылезаем из цикла count += 1 print(f"Ты ввёл {count} слов")

Грабли

5 типовых ошибок

1. ♾️ Забыл счётчик

В теле нет n += 1 — и Python крутится вечно. Решение: Ctrl+C и проверить, где меняется переменная условия.

2. ⚖️ = вместо ==

n = 5 — присваивание. n == 5 — сравнение. В условии цикла — всегда с двумя знаками.

3. 🛡️ Скобки из C++

while (n <= 5): — не ошибка, но не по-питоновски. Пишем без скобок: while n <= 5:.

4. 🤔 n += 1 непонятно

Это то же, что n = n + 1. Сокращённая форма, не магия.

5. 🤷 while, где нужен for

«Пробежать 10 раз» — это for i in range(10), не while i < 10. Знаешь число повторений → всегда for.
