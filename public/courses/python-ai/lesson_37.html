Что такое PyGame Zero?

🎮 Тренажёр для игр

Надстройка над «взрослым» PyGame. Убирает 80% служебного кода. Создан специально для того, чтобы новичок за 10 минут увидел результат на экране.

  • Готовый игровой цикл — пишешь только draw() и update()
  • Окно запускается само
  • Спрайты через Actor('имя') — находятся в папке images/
PyGame Zero vs PyGame
# 😴 PyGame (взрослый) — 25 строк
import pygame
pygame.init()
screen = pygame.display.set_mode((800,600))
clock = pygame.time.Clock()
while True:
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            pygame.quit(); exit()
    screen.fill((135,206,250))
    pygame.display.flip()
    clock.tick(60)

# ⚡ PyGame Zero — 3 строки
WIDTH, HEIGHT = 800, 600
def draw():
    screen.fill('skyblue')

Игровой цикл: 60 FPS

PyGame Zero 60 раз в секунду спрашивает программу: «что нарисовать?» и «что обновить?»

🎨 draw() — рисует

Только картинки, текст, фон. Никаких расчётов и изменений!

🤔 update() — думает

Двигает, считает, проверяет. Никаких рисований!

«draw() и update() — они дружат, но не лезут в дела друг друга»— золотое правило PyGame Zero

🎉 Окно открылось — первая победа! Мы официально начали делать игры.

Что произошло?

  • WIDTH и HEIGHT — магические имена, PyGame Zero их знает и создаёт окно такого размера
  • def draw() — функция с таким именем будет вызвана 60 раз в секунду
  • screen — готовый объект экрана, никакого import не нужно
  • 'skyblue' — цвет по имени. Работают все CSS-имена: tomato gold lavender
⚠️ Запуск — ТОЛЬКО через терминал

Не F5, не «Run». Пишем pgzrun game.py. Команда pgzrun знает игровой цикл.

🎭 Actor — волшебная коробка

Actor знает свою картинку, свои координаты и свой угол. Управляем им через точку.

game.py
WIDTH = 800
HEIGHT = 600

# создаём Actor — картинка basket.png из images/
basket = Actor('basket')
basket.pos = (400, 520)   # x=400, y=520

def draw():
    screen.fill('skyblue')
    basket.draw()   # рисуем корзинку

🧭 Система координат

Точка (0, 0) — верхний левый угол. Это не как на уроке математики!

x

0 → слева направо → 800

y

0 → сверху вниз → 600

  • basket.pos = (400, 520) — центр корзинки
  • basket.x, basket.y — отдельные координаты
  • basket.angle — поворот в градусах
«Actor — это такая волшебная коробка: знает свою картинку, свои координаты, свой угол»
09

⚡ Оживляем корзинку

Добавляем функцию update() — теперь корзинка поедет вправо!

game.py — оживляем
WIDTH = 800
HEIGHT = 600

basket = Actor('basket')
basket.pos = (400, 520)
speed = 2            # скорость в пикселях за кадр

def draw():
    screen.fill('skyblue')
    basket.draw()

def update():
    # 60 раз в секунду → +2 к x
    basket.x += speed

🧮 Простая математика

speed2 пикселя / кадр
FPS60 кадров / сек
Итого120 пикселей / сек
⚠️ Правило чистоты

Логика — ТОЛЬКО в update(). В draw() только рисование. Если смешаешь — игра будет глючить и тормозить.

basket.x += 2basket.y -= 5basket.angle += 1

🐛 Топ-5 ошибок первого урока

ModuleNotFoundError: pgzero

Установили в не ту версию Python. Проверь: python --version. Если 3.12 — поставь py -3.11 -m pip install pgzero.

pgzrun: command not found

Папка Python не в PATH. Запускай через python -m pgzero game.py.

No such file: basket.png

Проверь регистр! Basket.png ≠ basket.png. И что файл лежит в images/.

Спрайт огромный/крошечный

Kandinsky выдаёт 1024×1024. Открой в Paint → изменить размер до 200×200 → сохрани.

Корзинка едет мимо экрана и пропадает

В update() basket.x += 2 увеличивается бесконечно. В следующем уроке поставим ограничение краями экрана (clamp).

⭐ Звёздная задача только для отчаянных

Сделай так, чтобы корзинка плавно качалась влево-вправо — как маятник. Забегаем вперёд на урок 17–20.

bonus.py
import math

basket = Actor('basket')
basket.pos = (400, 520)

time = 0   # время в кадрах

def update():
    global time
    time += 1
    # синус качает корзинку
    basket.x = 400 + math.sin(time * 0.05) * 100

🌊 Что происходит?

  • math.sin() — функция-волна: даёт числа от −1 до 1, плавно
  • Умножаем на 100 → получаем от −100 до +100 пикселей
  • Прибавляем к 400 → центр колебания
  • time * 0.05 — скорость волны
💡 Правило: global нужно, чтобы менять переменную за пределами функции. Внутри функции просто time += 1 создало бы новую локальную.
