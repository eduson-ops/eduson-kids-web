Красивый проект — это не только код. Это порядок в файлах, понятные имена, папки для картинок и звуков. Возьми за привычку.

Начинаем с пустого окна

Правило профессионалов: сначала бегаем пустой фигурой, потом добавляем графику. Если работает логика на квадратах — она будет работать и на спрайтах.

    tick += 1
    if game_state == 'playing':
        update_game()
Слайд 09 · Dino

Герой, прыжок и гравитация

dino = Actor('dino_run1', (100, GROUND_Y))
dino.vy = 0                 # вертикальная скорость
dino.on_ground = True
RUN_FRAMES = ['dino_run1', 'dino_run2', 'dino_run3']

def on_key_down(key):
    if key == keys.SPACE:
        if game_state == 'menu':
            start_game()
        elif game_state == 'playing' and dino.on_ground:
            dino.vy = -14
            dino.on_ground = False
            sounds.jump.play()

def update_game():
    # гравитация
    dino.vy += 0.7
    dino.y += dino.vy
    if dino.y >= GROUND_Y:
        dino.y = GROUND_Y
        dino.vy = 0
        dino.on_ground = True
    # анимация бега
    if dino.on_ground:
        dino.image = RUN_FRAMES[(tick // 6) % 3]
    else:
        dino.image = 'dino_jump'

🧪 Что тут происходит

  • vy = -14 — импульс вверх (минус — потому что y=0 сверху)
  • vy += 0.7 — гравитация каждый кадр
  • on_ground защищает от бесконечного прыжка
  • tick // 6 — смена кадра каждые 6 тиков = ~10 fps анимации
✅ Паттерн знакомый? Да — он целиком из урока 42. Сегодня мы его применяем.
Слайд 10 · Препятствия

Кактусы: список + спавн + коллизии

import random

cacti = []                           # список препятствий
SPEED = 5                            # скорость движения мира

def spawn_cactus():
    kind = random.choice(['cactus_small', 'cactus_big'])
    cacti.append(Actor(kind, (WIDTH + 30, GROUND_Y)))

clock.schedule_interval(spawn_cactus, 1.5)    # новый кактус каждые 1.5 сек

def update_game():
    global score
    score += 1                       # счёт = пройденное расстояние

    # двигаем все кактусы влево, удаляем улетевшие и проверяем столкновение
    for c in cacti[:]:                # идём по копии — безопасно удалять
        c.x -= SPEED
        if c.x < -50:
            cacti.remove(c)
        elif dino.colliderect(c):
            sounds.hit.play()
            game_over()

🔁 Копия списка

Итерируемся по cacti[:], чтобы безопасно вызывать cacti.remove() внутри цикла.

🎲 Случайность

random.choice — без неё все кактусы одинаковые. А мы хотим игру, а не тир.

💥 Коллизия

dino.colliderect(c) — та же функция, что с яблоками в уроке 39. Переиспользуй.

Слайд 11 · Состояния

Меню → игра → gameover → меню

def start_game():
    global game_state, score
    game_state = 'playing'
    score = 0
    cacti.clear()                     # обнуляем мир
    dino.pos = (100, GROUND_Y)
    dino.vy = 0
    music.play('bg')
    music.set_volume(0.3)

def game_over():
    global game_state
    game_state = 'gameover'
    music.stop()

def on_key_down(key):
    if key == keys.SPACE:
        if game_state in ('menu', 'gameover'):
            start_game()
        elif game_state == 'playing' and dino.on_ground:
            dino.vy = -14
            dino.on_ground = False
            sounds.jump.play()

def draw_gameover():
    draw_game()                       # показываем последний кадр
    screen.draw.filled_rect(
        Rect(0, 0, WIDTH, HEIGHT),
        (0, 0, 0, 160)                # полупрозрачная пелена
    )
    screen.draw.text(f'SCORE: {score}',
        center=(WIDTH/2, HEIGHT/2 - 20),
        fontsize=60, color='white')
    screen.draw.text('Пробел — заново',
        center=(WIDTH/2, HEIGHT/2 + 40),
        fontsize=28, color=(6, 182, 212))

🔄 Карта состояний

menu→ SPACEplaying
playing→ hitgameover
gameover→ SPACEplaying

💡 Ключевые идеи

  • Перезапуск — это обнуление переменных, а не перезапуск программы
  • Один экран = одна ветка if/elif
  • На gameover — пелена поверх последнего кадра

Слайд 13 · Asteroids

Специфика Asteroids: вращение и выстрелы

import math

ship = Actor('ship', (WIDTH/2, HEIGHT/2))
ship.angle = 0         # градусы
bullets = []

def update_game():
    # поворот стрелками
    if keyboard.left:  ship.angle += 4
    if keyboard.right: ship.angle -= 4
    # движение вперёд по пробелу (или UP)
    if keyboard.up:
        rad = math.radians(ship.angle)
        ship.x += math.cos(rad) * 5
        ship.y -= math.sin(rad) * 5

    # движение пуль
    for b in bullets[:]:
        b.x += b.dx
        b.y += b.dy
        if not (0 < b.x < WIDTH and 0 < b.y < HEIGHT):
            bullets.remove(b)

def on_key_down(key):
    if key == keys.SPACE and game_state == 'playing':
        rad = math.radians(ship.angle)
        b = Actor('bullet', ship.pos)
        b.dx = math.cos(rad) * 10
        b.dy = -math.sin(rad) * 10
        bullets.append(b)
        sounds.shoot.play()

🧭 Углы и радианы

PyGame Zero вращает в градусах (actor.angle), а math.cos/sin хотят радианы. Поэтому math.radians(angle).

💥 Разбиение астероидов

for b in bullets[:]:
    for a in asteroids[:]:
        if b.colliderect(a):
            bullets.remove(b)
            asteroids.remove(a)
            if a.size > 1:
                split(a)
            sounds.boom.play()
            break

Слайд 16 · Дебаг

Топ-5 ошибок, которые тебя найдут

🪲 «Прыгает бесконечно»

Причина: забыл флаг on_ground или не сбросил vy = 0 при касании земли.

Решение: в update чётко if dino.y >= GROUND_Y: on_ground = True, vy = 0.

🪲 «Все кактусы исчезают разом»

Причина: for c in cacti: + cacti.remove(c) — ломает индексы.

Решение: итерация по копии — for c in cacti[:].

🪲 «Звук не играет»

Причина: файл не в папке sounds/, неправильное расширение или указан .wav в коде.

Решение: имя без расширения — sounds.jump.play().

🪲 «Рестарт оставляет старые объекты»

Причина: забыл cacti.clear() / bullets.clear() в start_game.

Решение: всё, что набирается за игру — чисти при старте.

🪲 «ИИ дал код — не работает»

Причина: YandexGPT сгаллюцинировал функцию, которой нет в PyGame Zero (например actor.rotate()).

Решение: проверяй в документации, не верь ИИ на слово. Это нормальный рабочий приём.
