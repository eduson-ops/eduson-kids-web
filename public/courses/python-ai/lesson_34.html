Понимаем

Что делает ИИ-ассистент

Генерация кода, подсказки, документация. Помощник — не замена.

Умеем

Формулировать промпт

Чёткий вопрос → полезный ответ. Мусор на входе → мусор на выходе.

Проверяем

Код ИИ — запуском

Никогда не доверяем слепо. Запустил → работает → принял.

Метафора

ИИ-ассистент — это друг-старшеклассник

Что умеет

  • Объяснить ошибку простыми словами
  • Сгенерировать шаблон функции
  • Придумать имена для героев игры
  • Найти документацию с ссылкой
  • Предложить 3 варианта решения

Иногда ошибается

  • Выдумывает методы, которых нет
  • Использует устаревший синтаксис
  • Ошибается в импортах
  • Пишет уверенно даже когда не знает
  • Это называется «галлюцинация»

Ты всё равно главный. ИИ — подмастерье.

Главная формула

Формула промпта: КРФЗ

Четыре буквы. Запомни, как таблицу умножения.

К
Контекст
«Я учусь Python, мне 12 лет»
Р
Роль
«Ты — добрый учитель программирования»
Ф
Формат
«Ответ до 5 строк, сначала объяснение»
З
Задача
«Объясни ошибку IndexError»

Запомни: КРФЗ — четыре слова, которые превращают ИИ в суперпомощника.

Пример полного промпта

Собираем всё вместе

# Контекст
Я учусь Python 3 месяца, мне 12 лет.
Работаю с проектом «Тамагочи» — ООП.

# Роль
Ты — дружелюбный учитель программирования
для школьников. Говори простым языком.

# Задача
Объясни, почему в коде возникает ошибку
"TypeError: unsupported operand type(s) for +"
при попытке сложить строку и число.

# Формат
1. Сначала объяснение в 2-3 предложениях
2. Потом короткий пример (до 5 строк)
3. Потом — как исправить
Без воды, без повторов.

↑ Вот это и есть «хороший промпт». Попробуйте сами переписать какой-нибудь свой вопрос по этой схеме.

Задача 3 · разбор

Функция is_prime(n)

1. Промпт по формуле КРФЗ
Markdown
Я учусь Python, мне 12 лет. Ты — дружелюбный учитель. Напиши функцию is_prime(n) — возвращает True, если n простое число, иначе False. Формат: код с комментариями, до 10 строк. Не забудь про 0, 1, 2.
2. Типичный ответ ИИ
Python
def is_prime(n): if n < 2: return False for i in range(2, n): if n % i == 0: return False return True print(is_prime(7)) # True print(is_prime(10)) # False

3. Правило: запусти и проверь!

Тестируй на: 0, 1, 2, 3, 4, 7, 10, 97. Все ответы правильные? Код твой. Ошибка? Продолжай итерировать.

Осторожно

Галлюцинации — когда ИИ врёт

ИИ генерирует правдоподобное, а не обязательно правдивое. Он уверенно пишет то, чего нет.

Тип 1

Несуществующий метод

mylist.magic_sort()
# AttributeError

Метода нет. ИИ его выдумал.

Тип 2

Устаревший синтаксис

print "Привет"
# Python 2, уже
# не работает

ИИ обучен на старом коде.

Тип 3

Неверный импорт

from pandas import json
# ImportError
# никогда не было

Модуль есть, сабмодуля нет.

Защита: запусти код. Не работает — проверяй каждую строку. Сомневаешься в методе — ищи в документации через Perplexity.

Чего не делать

Анти-паттерны при работе с ИИ

Копипаст без проверки

Скопировал код ИИ → вставил в проект → не запустил → сдал. Тройка гарантирована.

Слишком короткие промпты

«помоги с кодом» — мусор. Нужен КРФЗ: контекст + роль + задача + формат.

Слепое доверие

«ИИ сказал — значит правда». Нет! Каждый ответ проверяй: запуском, документацией, здравым смыслом.

Путать инструменты

Perplexity — факты со ссылками. YandexGPT — генерация и объяснение. Не мешай.

Workflow

Как проверять код ИИ

1

Прочитай

Понимаешь каждую строчку? Если нет — попроси объяснить.

2

Запусти

В IDE. Любой код, который пишет ИИ — проверяй запуском.

3

Тестируй

Крайние случаи: 0, 1, пустой список, большие числа.

4

Пойми

Сможешь объяснить код другу? Только тогда он по-настоящему твой.

Если ИИ дал код — запусти его. Работает? Хорошо. Нет? Разбирайся.

Сложности · Решения

Что может пойти не так

Проблема

Копирует код без проверки

→ Правило: запусти → убедись → только потом прими.

Проблема

Короткие промпты без контекста

→ Тренируем длинные. Считаем слова в промпте, цель — больше 20.

Проблема

Слепо верит ИИ

→ Каждый ответ — вопрос «а ты проверил?».

Проблема

Путает инструменты

→ Шпаргалка: Perplexity = факты + ссылки, YandexGPT = генерация + объяснение.

Частый вопрос

«Зачем учиться, если ИИ всё пишет?»

→ ИИ ошибается. Ты должен уметь чинить. Без знаний ты даже промпт не составишь. ИИ — инструмент, как молоток. Молоток забивает гвозди, но только если плотник знает, куда.
