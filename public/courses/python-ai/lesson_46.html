Паттерн #1 · списки объектов

Не плоди переменные — храни в списке

В игре всегда много одинаковых штук: враги, пули, монетки, препятствия.

Плохо: enemy1, enemy2, enemy3 — 100 переменных писать?

Хорошо: один список enemies = []. Его и наполняем, и чистим, и обходим.

⚠️
Не забудь [:]
Нельзя менять список, пока по нему идёшь. Идём по копии enemies[:] — иначе баг с пропадающими элементами.
main.py · шаблон из урока 39
import random

enemies = []

def spawn_enemy():
    x = random.randint(50, WIDTH - 50)
    enemies.append(Actor('enemy', (x, 0)))

clock.schedule_interval(spawn_enemy, 1.0)

def update():
    for e in enemies[:]:          # копия!
        e.y += 3
        if e.y > HEIGHT:
            enemies.remove(e)
        elif hero.colliderect(e):
            enemies.remove(e)
            score += 1
Паттерн #2 · состояния

Одна строка делает три экрана

Игра — это состояние + данные. Состояние определяет, что рисовать и что обновлять.

menu
→
playing
→
gameover
↺

На клавишу SPACE — меняем состояние. Это всё.

main.py
game_state = 'menu'
score = 0
lives = 3

def draw():
    screen.clear()
    if game_state == 'menu':
        screen.draw.text('SPACE — играть', center=(400,300))
    elif game_state == 'playing':
        hero.draw()
        for e in enemies: e.draw()
        screen.draw.text(f'Очки: {score}', (10,10))
    elif game_state == 'gameover':
        screen.draw.text('Конец! SPACE — заново', center=(400,300))
Паттерн #3 · коллизии

Столкновения через colliderect

Любая игра = прямоугольники, которые врезаются. colliderect возвращает True, если прямоугольники пересекаются.

  • Проверяй с конкретным объектом, не со списком
  • После попадания — удали или обработай, иначе срабатывает 60 раз/сек
  • Не забывай про else, когда нужно
main.py · коллизия из урока 39
def update():
    if game_state != 'playing':
        return

    for enemy in enemies[:]:
        enemy.x -= 4
        if hero.colliderect(enemy):
            enemies.remove(enemy)
            lives -= 1
            sounds.hit.play()
            if lives == 0:
                game_state = 'gameover'

Как спрашивать ИИ · пример

Код + ошибка + вопрос = ответ

Вот мой код: for apple in apples: if apple.y > 600: apples.remove(apple) Яблоки пропадают пачками, иногда на экране ни одного. Почему?
Ты меняешь список apples, пока идёшь по нему. Python пропускает элементы — отсюда «пачки». Решение: итерируйся по копии apples[:]: for apple in apples[:]: if apple.y > 600: apples.remove(apple) Так список в цикле «замораживается», а remove правит настоящий.
Ого, заработало! Спасибо.

Что хорошего в этом запросе

  • Код приведён целиком, не кусочек
  • Описано, что видно — «пачками»
  • Спросил «почему», а не «напиши»
  • Сам применил правку и проверил

Типовые грабли

Если что-то не работает — сверься

🐞

Игра тормозит

Объекты копятся, не удаляются. Добавь условие if e.y > HEIGHT: enemies.remove(e).

💥

Объекты пропадают пачками

Забыт [:]. Цикл: for e in enemies[:], а не enemies.

🎯

Коллизия не срабатывает

Пишешь hero.colliderect(enemies)? Надо с одним: hero.colliderect(enemy).

🔇

Звук не играет

В коде имя без .wav. Формат — .wav/.ogg. Файл в папке sounds/.

🔁

При рестарте старые враги остались

Забыл enemies.clear() в функции рестарта. Или обнули переменные вручную.

🤖

ИИ дал код, который не работает

Галлюцинация или устаревшая версия. Отличный учебный момент — проверь каждую строку вручную.

⭐ Звёздная задача

Частицы — вау-эффект за 15 строк

Маленькие точки, разлетающиеся при взрыве или столкновении. Этого не было в курсе — это твой плюс сверху.

Идея

  • Список словарей: {x, y, vx, vy, life}
  • При столкновении — создаём 10 частиц
  • В update сдвигаем, life -= 1
  • В draw рисуем filled_circle
🌟
Плюс на защите
«Я вышел за курс и сделал фичу, которой не учили» — любимая фраза преподавателей.
particles.py
import random

particles = []

def explode(x, y):
    for _ in range(12):
        particles.append({
            'x': x, 'y': y,
            'vx': random.uniform(-3, 3),
            'vy': random.uniform(-3, 3),
            'life': 30,
        })

def update():
    for p in particles[:]:
        p['x'] += p['vx']
        p['y'] += p['vy']
        p['life'] -= 1
        if p['life'] <= 0:
            particles.remove(p)

def draw():
    for p in particles:
        screen.draw.filled_circle(
            (p['x'], p['y']), 3, 'yellow')
