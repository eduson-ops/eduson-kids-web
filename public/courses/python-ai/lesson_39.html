Почему один объект — в списке

Плохо: 10 переменных

Python

Добавить 11-е яблоко = переписать полкода.

Хорошо: один список

apples = [] # добавить — одна строка apples.append(Actor('apple_red', (x, 0))) # обновить ВСЕ — один цикл for apple in apples: apple.y += 3

Теория · часть 2

Что такое colliderect

Каждый Actor в PyGame Zero — это не только картинка, но и невидимый прямоугольник вокруг неё.

colliderect отвечает на один вопрос:

Пересекаются ли две рамки прямо сейчас?

Возвращает True или False. Больше ничего. Ни скорости, ни точки пересечения.

Аналогия: две бумажки на столе. Если уголки накладываются хоть на пиксель — столкновение есть.

Совместный код · 15–30 мин

Шаг 1. Заводим список яблок

Python
import random WIDTH = 800 HEIGHT = 600 basket = Actor('basket', (400, 520)) apples = [] # ← вместо одной переменной — пустой список def spawn_apple(): x = random.randint(50, WIDTH - 50) new_apple = Actor('apple_red', (x, 0)) apples.append(new_apple) def update(): for apple in apples: apple.y += 3 def draw(): screen.fill('skyblue') basket.draw() for apple in apples: apple.draw()

Функция spawn_apple создаёт ОДНО яблоко и добавляет в список. Пока что вызывать её надо вручную — об этом на слайде 11.

Speaker notes

Пиши код вместе с ребёнком, не копируй. Самые частые ошибки здесь: забыли import random; написали apples = Actor(...) вместо apples = []. Запусти — пока ничего не видно, потому что спавн не вызывается. Это мостик к следующему слайду.

Совместный код · 30–45 мин

Шаг 2. Ловим яблоко — colliderect

Python
def update(): for apple in apples[:]: # ← ВАЖНО: копия списка! apple.y += 3 if basket.colliderect(apple): apples.remove(apple) # поймали — убрали # тут потом будет score += 1 elif apple.y > HEIGHT: apples.remove(apple) # улетело — тоже убрали

Запомни синтаксис

basket.colliderect(apple) — один конкретный объект в скобках, не весь список.

Частая ошибка: basket.colliderect(apples) — PyGame Zero попытается сравнить корзину со списком и упадёт.

Две причины убрать

  • Поймали — столкновение с корзиной
  • Упало — ушло ниже экрана (иначе копятся и тормозят)

Speaker notes

Здесь два подводных камня: (1) передача всего списка вместо объекта, (2) отсутствие [:] — о нём отдельный слайд. Не объясняй [:] сейчас, просто скажи «сейчас я покажу, почему мы пишем двоеточие в квадратных скобках».

Главный лайфхак урока

Почему мы пишем apples[:]

Без копии — БАГ

Python
for apple in apples: if basket.colliderect(apple): apples.remove(apple) # некоторые яблоки «пропустятся» # Python перестаёт понимать, где он

Ты идёшь по списку гостей и одновременно вычёркиваешь их из этого же списка. Номера сбиваются.

С копией — ОК

Python
for apple in apples[:]: if basket.colliderect(apple): apples.remove(apple) # копия не меняется # оригинал меняется спокойно

apples[:] — это «снимок» списка. Идём по снимку, удаляем из оригинала.

«Нельзя менять таблицу, пока по ней идёшь»

Speaker notes

Это самый важный и самый «взрослый» концепт урока. Если дети не поняли — сделай демку без [:], выведи print(len(apples)), покажи, что на глазах пропадает не ВСЁ что должно. Потом добавь [:] — всё чинится. «Вау-эффект» закрепит знание.

Усложнение · 45–55 мин

Шаг 3. Яблоки появляются сами

Python
def spawn_apple(): x = random.randint(50, WIDTH - 50) apples.append(Actor('apple_red', (x, 0))) # запускаем таймер ОДИН раз при старте игры clock.schedule_interval(spawn_apple, 1.0) # раз в секунду PyGame Zero сам вызовет spawn_apple()

schedule

Вызвать функцию один раз через N секунд.

schedule_interval

Вызывать каждые N секунд, пока игра работает.

unschedule

Остановить таймер (понадобится в уроке 40 для паузы).

clock — встроенный объект PyGame Zero. Импортировать не нужно. Он уже есть в твоей программе.

Speaker notes

Разведи schedule и schedule_interval — это два разных таймера. Первый стреляет один раз, второй тикает. Покажи на пальцах: schedule — будильник, schedule_interval — часовые удары.

Полный код урока 39

Всё вместе — 35 строк

import random

WIDTH = 800
HEIGHT = 600

basket = Actor('basket', (400, 520))
apples = []

def spawn_apple():
    x = random.randint(50, WIDTH - 50)
    apples.append(Actor('apple_red', (x, 0)))

def on_mouse_move(pos):
    basket.x = max(50, min(WIDTH - 50, pos[0]))

def update():
    for apple in apples[:]:
        apple.y += 3
        if basket.colliderect(apple):
            apples.remove(apple)
        elif apple.y > HEIGHT:
            apples.remove(apple)

def draw():
    screen.fill('skyblue')
    basket.draw()
    for apple in apples:
        apple.draw()

clock.schedule_interval(spawn_apple, 1.0)

Speaker notes

Покажи этот слайд как «контрольную точку». Попроси ребёнка мышкой ткнуть в строку, которую НЕ понимает, и разбери именно её. Не трать время на то, что уже ясно.

Жизненный цикл яблока

От появления до исчезновения

01 · SPAWN

Родилось

Таймер вызвал spawn_apple. Actor создан, добавлен в список, y = 0.

02 · UPDATE

Падает

Каждый кадр apple.y += 3. 60 раз в секунду.

03 · CHECK

Проверка

colliderect с корзиной? Или y > HEIGHT?

04 · REMOVE

Удалено

apples.remove(apple). Actor исчезает с экрана.

05 · NEXT

Новое

Через секунду таймер создаст другое яблоко.

Поэтому в списке всегда разное число элементов. Это динамика.

Speaker notes

Визуализируй цикл жирной стрелкой: spawn → падение → удаление. Спроси ребёнка: «А если я не буду удалять улетевшие?». Ответ: список будет расти бесконечно, игра замедлится. Это мотивирует строку elif apple.y > HEIGHT.

Сложности и решения

Три классических бага

Пропадают все сразу

Забыли [:]. Или использовали for i, apple in enumerate(apples): apples.pop(i).

Фикс: for apple in apples[:]

Игра тормозит

Яблоки улетают за экран, но из списка не удаляются. Через 2 минуты — 120 штук.

Фикс: ветка elif apple.y > HEIGHT

colliderect не работает

Написано basket.colliderect(apples) — со всем списком вместо одного яблока.

Фикс: внутри цикла, с apple

Диагностика: добавь print(len(apples)) в update. Если число растёт бесконечно — не чистим улетевшие.

Speaker notes

Все три бага встречаются у 70% детей. Заранее держи слайд открытым — можно показать сразу, не тратя время на поиск. print(len(apples)) — магический приём отладки, покажи обязательно.

Библиотека random · вспомнили урок 21

Случайность — это двигатель игр

Python
import random random.randint(50, 750) # случайное целое от 50 до 750 включительно random.randint(2, 5) # скорость от 2 до 5 пикселей за кадр random.choice(['red', 'green', 'gold']) # случайный элемент списка random.random() # дробное от 0.0 до 1.0

В нашей игре

  • Случайный x для появления яблока
  • Случайная скорость падения (для ДЗ)
  • Случайный вид яблока (тоже в ДЗ)

Запомни

randint(a, b) — оба конца включительно. От 1 до 6 — это кубик, шестёрка тоже выпадет.

Speaker notes

Дети часто удивляются: «А random это не „магия“?». Объясни, что это математический алгоритм, который даёт числа, похожие на случайные. Для игр этого хватает. Компьютер «бросает кубик» 1000 раз в секунду.
