Два главных свойства

Что делает множество особенным

🎯

1. Уникальность

Каждый элемент встречается ровно один раз. Повтор добавляется — но ничего не меняется.

s = {1, 2, 2, 3, 3, 3}
print(s)   # {1, 2, 3}
🔀

2. Нет порядка

Индексации не существует. Обращение `s[0]` — ошибка.

s = {"a", "b", "c"}
print(s[0])
# TypeError: 'set' object is
# not subscriptable

Синтаксис

Как создать множество

Три способа

  • Фигурные скобки с элементами: {1, 2, 3}
  • Из списка: set([1, 2, 2, 3])
  • Пустое: set() ← не {}
⚠️
Ловушка!
{} — это пустой словарь, а не множество. Запомни: пустое множество — только set().
sets_intro.py 
letters = {"a", "b", "c"}
numbers = {1, 2, 3, 4, 5}

# из списка — автоматически уберёт повторы
nums = set([1, 1, 2, 3, 3])
print(nums)       # {1, 2, 3}

# ПУСТОЕ множество
empty_set = set()
print(type(empty_set))   # <class 'set'>

# А это УЖЕ СЛОВАРЬ
not_a_set = {}
print(type(not_a_set))   # <class 'dict'>
4 метода

Клади и вынимай: add · remove · discard · clear

➕

add(x)

Добавить в мешок. Если уже есть — не удваивает.

s.add("🍎")
❌

remove(x)

Убрать. Если нет — KeyError!

s.remove("🍌")
# KeyError
🤫

discard(x)

Тоже убрать. Но если нет — молчит.

s.discard("🍌")
# OK, тишина
🧹

clear()

Высыпать весь мешок. Остаётся пустое множество.

s.clear()
print(s)  # set()
«remove() падает, если элемента нет. discard() — нет, он молчит.» — фраза преподавателя #9
Совместный кодинг · 15–25 мин

Мини-демо: угаданные буквы

Пишем вместе с учеником.
Главное — увидеть, что повторы не лезут в множество.

  • Создаём пустое множество: guessed = set()
  • Просим у пользователя букву: input()
  • Кладём в множество: guessed.add(letter)
  • Выводим guessed — смотрим, что повторов нет
  • Останавливаем по слову "стоп"
guessed_letters.py 
guessed = set()

while True:
    letter = input("Введи букву (или 'стоп'): ")
    if letter == "стоп":
        break
    guessed.add(letter)
    print(f"Угадано уже: {guessed}")

print(f"Всего уникальных: {len(guessed)}")

Три главные операции

Объединение · пересечение · разность

| Объединение. Всё, что есть хотя бы в одном множестве. a | b
& Пересечение. Только то, что есть в обоих. a & b
− Разность. То, что есть в a, но нет в b. a - b
💡
Лайфхак преподавателя
| — «два класса собрали вместе». & — «дружба двух компаний». - — «кто есть у меня, но не у тебя».

Самостоятельная задача · 25–45 мин

«Два класса в кино»

🎬 Условие

В кино приехали два класса — 6А и 6Б. Кто-то был и там, и там (перешли из класса в класс). Нам нужно ответить на три вопроса:

  • Кто был в обоих классах?
  • Кто был хотя бы в одном?
  • Кто был только в 6А?
💭
Подсказка
Каждый ответ — это одна операция над двумя множествами.
cinema.py 
a = {"Саша", "Катя", "Петя"}
b = {"Катя", "Лена", "Миша"}

# 1. Кто был в обоих?
print("В обоих:", a & b)
# {'Катя'}

# 2. Кто был хотя бы в одном?
print("Всего:", a | b)
# {'Саша', 'Катя', 'Петя',
#  'Лена', 'Миша'}

# 3. Кто был только в 6А?
print("Только 6А:", a - b)
# {'Саша', 'Петя'}
Усложнение · 45–55 мин

Лайфхак: убрать дубли в одну строку

Самый знаменитый трюк Python: оборачиваем список в set() и сразу обратно в list() — дубликатов нет.

📋 list
→
🎒 set
→
📋 list без дублей
⚠️
Минус подхода
Теряется порядок. Если он важен — нужен другой приём (через словарь, 2 урока спустя).
dedup.py 
names = ["Аня", "Вова", "Аня", "Маша",
         "Вова", "Коля", "Маша", "Аня"]

# один проход — и готово!
unique = list(set(names))

print(unique)
# ['Аня', 'Маша', 'Вова', 'Коля']
# (порядок может отличаться)

print(f"Было {len(names)}, стало {len(unique)}")
# Было 8, стало 4

Проверка наличия

Оператор in — мгновенная проверка

«Эта буква уже угадана?» — это letter in guessed. Возвращает True или False.

«Проверка if name in users работает в множестве ОЧЕНЬ быстро.»— фраза преподавателя #8
  • in в списке — перебор всех элементов
  • in в множестве — мгновенно, даже на 1 000 000 элементов
  • Идеально для Виселицы и других проверок
check_in.py 
guessed = {"а", "б", "к"}

letter = input("Проверяем букву: ")

if letter in guessed:
    print("⚠️ Уже угадывал!")
else:
    print("✅ Новая буква — добавляю")
    guessed.add(letter)

print(f"Сейчас: {guessed}")

Как объяснять на уроке

«Множество — мешок, в котором каждой вещи ровно по одной».
«Порядка в мешке нет, не ищи индекс — его не существует».
«Пустое множество — set(), а не {} — иначе получится словарь».

«- — разность, кто есть у меня, но не у тебя».
«Хочешь убрать дубли? Оберни в set() и обратно в list()».

Сложности и решения

5 ловушек и как их обойти

🚫

set = {} — не работает

Это словарь. Решение: только set() для пустого.

🔢

guessed[0] — ошибка

У множества нет порядка. Решение: либо цикл for, либо преобразуй в list.

🔀

Путают | с or

Оператор | — это НЕ логическое «или». Решение: диаграммы Эйлера-Венна на бумаге.

💥

remove vs discard

Не понимает разницу. Решение: показать падение remove на отсутствующем элементе.

🎒

Забывает, что множество изменяемое

В отличие от кортежа, множество можно менять. Напоминаем: мешок открыт, класть и вынимать можно.

Звёздная задача ⭐

Сколько уникальных букв в твоей любимой книге?

Берём первый абзац, приводим всё к нижнему регистру, оборачиваем в множество — и len().

  • Скопировать первый абзац
  • Сохранить в переменную text
  • text.lower() — все буквы маленькие
  • set(text) — уникальные символы
  • len(...) — сколько их
🚀
Мини-вызов
Отфильтруй пробелы и знаки препинания — оставь только буквы.
unique_letters.py 
text = """Жили-были старик со старухой
у самого синего моря. Они жили в ветхой
землянке ровно тридцать лет и три года."""

letters = set(text.lower())
print(f"Всего символов: {len(text)}")
print(f"Уникальных: {len(letters)}")
print(letters)

# только буквы:
only_letters = {c for c in text.lower() if c.isalpha()}
print(f"Только букв: {len(only_letters)}")
