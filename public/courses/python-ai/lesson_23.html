Инструмент 1

readlines() — список строк одной командой

Читает файл целиком и возвращает список строк. Одна строка файла = один элемент списка.

Удобно, когда нужно пройти по строкам циклом или посчитать количество.

⚠️
Символ \n остаётся!
Каждый элемент заканчивается переводом строки. Не забывай .strip() при выводе.
notes_read.py
Python
with open("notes.txt", "r", encoding="utf-8") as f: lines = f.readlines() print(lines) # ['купить молоко\n', # 'сделать уроки\n', # 'погулять\n'] print(len(lines)) # 3 строки print(lines[0].strip()) # купить молоко
Инструмент 2

writelines() — записать список в файл

Принимает список строк и записывает их по порядку.

❗
Не добавляет \n сам!
Если забыть — всё слипнется в одну строку. Всегда сам ставь перевод строки в конце каждой.

Совет: используй [x + "\n" for x in items] — короткий способ добавить перевод строки каждому элементу.

todo_write.py
Python
items = ["купить молоко", "сделать уроки", "погулять"] with open("todo.txt", "w", encoding="utf-8") as f: f.writelines([x + "\n" for x in items]) # файл todo.txt: # купить молоко # сделать уроки # погулять
Инструмент 3

Курсор в файле — мигающая палочка

В файле всегда есть «указка». tell() говорит — где она сейчас. seek(N) — переставь её в позицию N.

Привет, мир!

f.tell() → 5 · прочитали 5 байт, курсор стоит на шестом символе. f.seek(0) — вернулись в начало.

📍

f.tell()

Вопрос «где я стою?». Возвращает число — позицию в байтах.

🎯

f.seek(N)

Команда «иди в позицию N». seek(0) — в начало.

🧠

Зачем?

Перечитать файл несколько раз, прыгнуть в середину, найти нужное место.

Инструмент 3 · код

seek и tell — управляем курсором

cursor_demo.py
Python
with open("notes.txt", "r", encoding="utf-8") as f: print(f.tell()) # 0 — мы в начале chunk = f.read(5) # читаем 5 символов print(chunk) # "Приве" print(f.tell()) # 10 (кириллица = 2 байта) f.seek(0) # возвращаемся в начало print(f.read()) # читаем весь файл заново
ℹ️
Почему tell() после read(5) = 10?
Одна русская буква занимает 2 байта в UTF-8. 5 символов × 2 = 10 байт. На английских буквах будет ровно 5.
⚠️
seek(-1) в текстовом режиме
Отрицательные смещения в текстовом режиме запрещены. Нужны — переходи в rb.

Курсор — это суперспособность. Можно читать один и тот же файл несколько раз без повторного open.

Инструмент 4

Режимы открытия файла — карта

r

Чтение

Читаем существующий. Если файла нет — FileNotFoundError.

w

Запись

Стирает всё содержимое! Создаёт, если файла нет.

a

Дозапись

Дописывает в конец. Создаёт, если нет. Не стирает.

x

Создание NEW

Создаёт новый. Если уже есть — упадёт с ошибкой. Страховка.

b

Бинарный NEW

Добавляется к любому: rb, wb. Для картинок и музыки.

🔤

encoding="utf-8"

Обязательно в текстовых режимах. В бинарных — не нужен.

Бинарные файлы

rb / wb — для картинок и музыки

Текст = буквы. Картинка = байты.
Для не-текстовых файлов — добавляем букву b.

Читаем один файл в rb, пишем другой в wb. Получаем точную копию картинки, музыки или любого бинарного файла.

🚫
Не открывай картинку в r!
Получишь UnicodeDecodeError — Python попытается декодировать байты как UTF-8.
copy_image.py
Python
# копируем картинку целиком with open("cat.png", "rb") as src, \ open("cat_copy.png", "wb") as dst: dst.write(src.read()) print("Копия готова!") # открой cat_copy.png — та же кошка
Инструмент 5

try / except — ремень безопасности

Без try/except: программа падает при первой ошибке.
С ним: ловит ошибку и спокойно её обрабатывает.

Внутри try — код, который может сломаться. Внутри except — план Б.

💡
Ловите КОНКРЕТНО
Голый except: — плохой стиль. Пиши тип ошибки: except FileNotFoundError:.
🟢 try: код
→
💥 ошибка?
↳
🟡 except: план Б
🏁 программа продолжает жить
try / except в деле

Ловим ошибку конкретно

safe_read.py
Python
try: with open("no_such.txt", "r", encoding="utf-8") as f: print(f.read()) except FileNotFoundError: print("Файла нет, создам новый") with open("no_such.txt", "x", encoding="utf-8") as f: f.write("hello\n") except PermissionError: print("Нет доступа к файлу")

Частые «файловые» ошибки

  • FileNotFoundError — файла не существует
  • FileExistsError — режим x, а файл уже есть
  • PermissionError — нет прав на чтение / запись
  • UnicodeDecodeError — текстовый режим на бинарном файле
  • IsADirectoryError — это папка, а не файл

Можно ловить несколько типов — каждый своим блоком except.

Инструмент 6

Режим x — страховка от потери данных

«Создать файл, если его нет. Упасть, если уже есть».

Защищает от случайной перезаписи. В паре с try/except — идеальный способ инициализировать файл один раз.

🛡️
Когда пригодится
Первая настройка — создать settings.txt, save.json, progress.txt. Потом — уже не затирать.
init_file.py
Python
try: with open("progress.txt", "x", encoding="utf-8") as f: f.write("level=1\nscore=0\n") print("Создал новый файл прогресса") except FileExistsError: print("Файл уже есть — не трогаю")
Практика 1 · 5–8 мин

Пронумеровать строки файла

Задача: прочитать notes.txt через readlines(), вывести с номерами вида 1) купить молоко.

Важно: каждая строка заканчивается на \n. Используй .strip(), иначе вывод будет с двойным отступом.

  • readlines() даёт список
  • enumerate() — индекс + элемент
  • .strip() снимает \n и пробелы
numbered.py
Python
with open("notes.txt", "r", encoding="utf-8") as f: lines = f.readlines() for i, line in enumerate(lines, start=1): print(f"{i}) {line.strip()}") # 1) купить молоко # 2) сделать уроки # 3) погулять

Скопировать картинку бинарно

Задача: скопировать cat.png в cat_copy.png через rb/wb. Открыть копию — убедиться, что картинка отображается.

🔗
Два with одной строкой
Можно открывать несколько файлов сразу: with open(a) as x, open(b) as y:
⚠️
В rb/wb нет encoding
Бинарные файлы — это байты, у них нет кодировки. Параметр не нужен.

Python
with open("cat.png", "rb") as src, \ open("cat_copy.png", "wb") as dst: data = src.read() dst.write(data) print(f"Скопировано {len(data)} байт") # → открой cat_copy.png — это та же кошка

try: with open("todo.txt", "r", encoding="utf-8") as f: tasks = f.readlines() print(f"Задач: {len(tasks)}") for i, t in enumerate(tasks, 1): print(f"{i}) {t.strip()}") except FileNotFoundError: print("Файла нет, создаю пустой...") with open("todo.txt", "x", encoding="utf-8") as f: f.write("") print("Готово. Мини-проект · 10 мин

«Лог игры» — дневник запусков

Каждый запуск программы дописывает в game_log.txt строку со временем. Потом считает, сколько раз её запускали.

Всё, что мы учили сегодня — в одном проекте. Настоящий лог. Настоящая игра делает так же.

  • time.strftime — получить строку времени
  • open(..., "a") — дописать запись в лог
  • try / except — защита от первого запуска
  • readlines() + len — сколько раз запускали
📋
Формат строки лога
2026-04-20 14:32 — запущено. Шаблон: time.strftime("%Y-%m-%d %H:%M").
Мини-проект · код

«Лог игры» — собираем вместе

game_log.py
Python
import time stamp = time.strftime("%Y-%m-%d %H:%M") entry = f"{stamp} — запущено\n" # 1) Дописываем запись — режим 'a' сам создаст файл with open("game_log.txt", "a", encoding="utf-8") as f: f.write(entry) # 2) Считаем запуски — защищаемся от первого раза try: with open("game_log.txt", "r", encoding="utf-8") as f: lines = f.readlines() print(f"Игра запущена уже {len(lines)} раз(а)") print(f"Последний запуск: {lines[-1].strip()}") except FileNotFoundError: print("Лог ещё не создан — это первый запуск")

Частые ошибки

Что может сломаться и как чинить

🕳️

Голый except:

Ловит ВСЁ, включая твои баги. Лечим: пиши конкретный тип ошибки.

💥

Картинка в текстовом режиме

UnicodeDecodeError. Лечим: режим rb.

🚪

Забыли закрыть файл

Данные не сохранились. Лечим: всегда with.

↩️

seek(-1) в r

Ошибка. Лечим: либо переходи в rb, либо считай абсолютную позицию.

🗂️

«А где файл?»

Дети не знают, куда Python пишет. Лечим: открыли проводник, показали папку.

➖

writelines без \n

Всё слипается в одну строку. Лечим: [x + "\n" for x in items].
