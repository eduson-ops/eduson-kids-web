Теория · 5–15 мин

Методы с аргументами и возвратом

03
  • self — это «я сам», тот объект, у которого вызвали метод
  • other (или любой параметр) — другой объект, с которым работаем
  • return — метод возвращает значение, как обычная функция
  • Метод может менять другой объект: other.hp -= self.damage
Имена методов — глаголы: attack, heal, feed, sleep. Атрибуты — существительные: name, hp, damage.
Python
class Character: def attack(self, other): # self — кто бьёт # other — кого бьют other.hp -= self.damage return self.damage # сколько нанесли def is_alive(self): return self.hp > 0 # проверка состояния
Разбираемся

self vs параметр метода

04

self = «я сам»

Первый параметр любого метода. Это текущий объект, у которого вызвали метод.

hero.attack(goblin)
# внутри attack:
# self  -> hero
# other -> goblin

other = «другой»

Обычный параметр. Имя любое — other, enemy, target. Это объект, с которым self взаимодействует.

def attack(self, other):
    # self.damage — МОЙ урон
    # other.hp    — ЕГО здоровье
    other.hp -= self.damage
Топ-ошибка: забыть self. внутри метода и написать просто hp → NameError.
Совместный кодинг · 15–25 мин

Базовый класс Character

05
Python
class Character: def __init__(self, name, hp, damage): self.name = name self.hp = hp self.damage = damage def is_alive(self): return self.hp > 0 def attack(self, other): other.hp -= self.damage print(f"{self.name} бьёт " f"{other.name} на {self.damage}")

Что тут происходит?

  • __init__ — сохраняет имя, хп, урон
  • is_alive — возвращает True/False
  • attack — меняет чужой объект
Проверяй состояние методом, а не через hero.hp > 0 снаружи.
Наследование

Hero наследует Character

06
Python
class Hero(Character): def heal(self): self.hp += 20 print(f"{self.name} лечится, HP={self.hp}") hero = Hero("Маг", 100, 15) goblin = Character("Гоблин", 40, 5) print(hero.name, hero.damage) # Маг 15 hero.heal() # HP=120 hero.attack(goblin) # Маг бьёт Гоблина на 15

Главный цикл

Бой: объекты взаимодействуют в цикле

07
Python
hero = Hero("Маг", 100, 15) goblin = Character("Гоблин", 40, 5) while hero.is_alive() and goblin.is_alive(): hero.attack(goblin) if goblin.is_alive(): goblin.attack(hero) print("Победил:", hero.name if hero.is_alive() else goblin.name)
  • while крутится, пока оба живы
  • сначала атакует герой
  • если враг ещё жив — отвечает
  • каждый шаг меняет состояние объектов
Ловушка: если забыть if goblin.is_alive() — мёртвый враг продолжит бить.

Расширяем РПГ: класс Boss и волна врагов

08

1. Класс Boss(Character)

Усиленный урон, больше хп, особое сообщение при ударе.

2. Список врагов

enemies = [goblin, goblin2, boss]

3. Бой по очереди

for enemy in enemies: — пока герой жив, дерёмся со следующим.

class Boss(Character):
    def attack(self, other):
        other.hp -= self.damage * 2          # боссы бьют вдвое
        print(f"{self.name} ГРОМИТ {other.name}!")

enemies = [Character("Гоблин", 30, 5),
           Character("Орк", 50, 8),
           Boss("Дракон", 80, 10)]

for enemy in enemies:
    while hero.is_alive() and enemy.is_alive():
        hero.attack(enemy)
        if enemy.is_alive(): enemy.attack(hero)
    if not hero.is_alive(): break
Бонус

Полиморфизм: одно имя — разное поведение

09
Python
class Goblin(Character): def attack(self, other): other.hp -= self.damage print(f"{self.name} ржёт и бьёт на {self.damage}") class Boss(Character): def attack(self, other): other.hp -= self.damage * 2 print(f"{self.name} ГРОМИТ на {self.damage*2}") # магия: один цикл — разное поведение for enemy in [Goblin("Гобо",30,5), Boss("Дракон",80,10)]: enemy.attack(hero)

Идея

Метод называется одинаково — attack. Но у каждого класса своя реализация. Внешний код об этом не знает.

«Один класс = одна сущность». Не пихай героя и карту в один класс.
Тестируем · 45–55 мин

Ловим типичные баги

10

🐛 Бесконечный цикл

Причина: у врага damage=0 или проверка hp >= 0.

Фикс: while hp > 0, не >= 0.

🐛 Отрицательный HP

Причина: урон больше оставшегося хп.

Фикс: self.hp = max(0, self.hp - dmg).

🐛 NameError: hp

Причина: забыли self. в методе.

Фикс: self.hp -= ..., не hp -= ....

🐛 Метод ничего не возвращает

Причина: только print, нет return.

Фикс: print — для человека, return — для программы.

Проект «Банк»

16

Класс Account

attrname attrbalance defdeposit(amount) defwithdraw(amount) defshow() deftransfer(other, amount)
Метод transfer — тот же паттерн, что attack: один объект меняет другой.
Python
class Account: def __init__(self, name, balance=0): self.name = name self.balance = balance def deposit(self, amount): self.balance += amount def withdraw(self, amount): if amount <= self.balance: self.balance -= amount return True return False def transfer(self, other, amount): if self.withdraw(amount): other.deposit(amount) def show(self): print(f"{self.name}: {self.balance}")

Счёт

score = 0
hero = Hero("Маг", 100, 15)
enemies = [Character("Гоблин", 30, 5),
           Character("Орк",    50, 8),
           Boss("Дракон",      80, 10)]

for enemy in enemies:
    print(f"\n⚔️  Бой: {hero.name} vs {enemy.name}")
    while hero.is_alive() and enemy.is_alive():
        hero.attack(enemy)
        if enemy.is_alive():
            enemy.attack(hero)
    if hero.is_alive():
        score += 10
        print(f"✅ Победа! Очков: {score}")
        hero.heal()
    else:
        print("☠️  Герой пал")
        break

print(f"Итог: {score} очков")
Хорошие привычки

Чистый ООП-код с первого дня

18

✅ Делай так

  • Имена методов — глаголы (attack, heal)
  • Атрибуты — существительные (name, hp)
  • Один класс — одна сущность
  • Состояние проверяй методом (is_alive())
  • Методы и возвращают, и меняют состояние

❌ Не делай так

  • Не копайся в атрибутах извне: hero.hp > 0
  • Не смешивай героя и карту в один класс
  • Не забывай self. внутри метода
  • Не думай, что print = return
  • Не копируй совет ИИ, не разобравшись
