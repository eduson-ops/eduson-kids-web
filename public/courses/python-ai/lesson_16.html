Главная идея

Двумерный список — лист в клеточку

Обычный список — это полка с коробками. Двумерный — это шкаф: сначала выбираем полку (y), потом коробку на полке (x).

  • maze[0] — вся первая строка
  • maze[0][0] — левая верхняя клетка
  • maze[y][x] — сначала y, потом x
  • Как в шахматах: этаж, потом квартира
0 1 2 3 4 5 6 7 8 0 # # # # # # # # # 1 # P · · # · · · # 2 # · # · # · # · # 3 # · · · # · · E # 4 # # # # # # # # #

  • Зачем не просто строки?
    Чтобы можно было менять клетки: собрать монетку, сдвинуть игрока.

Печать карты: вложенный for

Двойной цикл

Где игрок? Двойной for

Проходим по всем клеткам карты и ищем букву P.

  • Внешний цикл — по строкам
  • Внутренний — по клеткам в строке
  • Нашли P — запомнили y и x
  • enumerate даёт номер + значение
find_player.py
Python
px, py = 0, 0 for y, row in enumerate(maze): for x, cell in enumerate(row): if cell == "P": px, py = x, y print(f"Игрок на x={px}, y={py}")
Управление

WASD: координаты в голове

W
A
S
D

Как в Minecraft / любой компьютерной игре

Что меняется в координатах

  • W — вверх → y = y − 1
  • S — вниз → y = y + 1
  • A — влево → x = x − 1
  • D — вправо → x = x + 1
🔄
Внимание!
W — это минус y. Сверху номер меньше!

Паттерн

Игровой цикл: while True + break

while True = «повторяй вечно».

break — «хватит, выходим».
continue — «пропусти, давай заново».

  • Игрок на E? → break + «Победа!»
  • Стена? → continue
  • Не та клавиша? → continue
game_loop.py
Python
steps = 0 while True: show_map() command = input("WASD: ") # ... движение ... steps += 1 if maze[ny][nx] == "E": print(f"🏁 Победа! {steps} шагов") break
Вся программа

Архитектура на салфетке

START
  │
  ├── maze = [ list("..."), list("..."), ... ]     # 1. карта
  │
  ├── px, py = найти "P"                              # 2. позиция игрока
  │
  ├── while True:                                     # 3. игровой цикл
  │       ├── show_map()
  │       ├── command = input()
  │       ├── если стена         ──► continue
  │       ├── если вне карты     ──► continue
  │       ├── двигаем P, steps += 1
  │       └── если E              ──► print("Победа!"), break
  │
END

maze_game.py
Python
maze = [ list("#########"), list("#P # #"), list("# # # # #"), list("# # E#"), list("#########"), ] px, py = 1, 1 steps = 0 def show_map(): for row in maze: print("".join(row)) while True: show_map() cmd = input("Ход (WASD): ").lower().strip() nx, ny = px, py if cmd == "w": ny -= 1 elif cmd == "s": ny += 1 elif cmd == "a": nx -= 1 elif cmd == "d": nx += 1 else: continue if maze[ny][nx] == "#": print("🧱 Стена!"); continue if maze[ny][nx] == "E": print(f"🏁 Победа! {steps+1} шагов"); break maze[py][px] = " " maze[ny][nx] = "P" px, py, steps = nx, ny, steps + 1

Формула промпта

  • Размер: «7 строк × 10 столбцов»
  • Легенда: что значит каждый символ
  • Только текст — без пояснений, без ```
  • Один вход, один выход (иначе найдёт нулевой путь)

Нарисуй ASCII-лабиринт 7 строк × 10 столбцов.
Символы:

пробел — проход
P — игрок (один)
E — выход (один)
Только сам лабиринт, без пояснений и без ```.
Должен быть ровно один путь от P к E.

Сложности и решения

Топ-5 ошибок учеников

1. Путают maze[y][x]

Пишут maze[x][y] — и всё ломается. Решение: листок в клеточку, рисуем точки по координатам.

2. Забывают проверку стены

Сначала двигают игрока, потом проверяют — поздно. Решение: правило «сначала nx, ny, потом if, потом двигаем».

3. Печать без join

print(row) выдаёт ['#','P',...]. Решение: сравниваем вывод print(row) и print("".join(row)).

4. IndexError на краю

Игрок подошёл к краю, ещё шаг — IndexError. Решение: 0 <= ny < len(maze) и то же для nx.

5. Карта от Алисы «косая»

Строки разной длины → IndexError в середине игры. Решение: глазами проверить, подровнять руками.
