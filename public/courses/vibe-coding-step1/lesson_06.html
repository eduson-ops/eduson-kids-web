Теория · метафора

Функция — это рецепт блинов

Один раз записал — пекёшь блины каждый день, не переписывая рецепт заново. Передал ингредиенты (параметры) — получил стопку блинов (возвращаемое значение).

def — «записать рецепт» (параметры) — ингредиенты return — результат вызов — «испеки!»
Словарь урока: функция, параметр, аргумент, возврат (return), вызов, декомпозиция. Хорошо зайдёт вопрос: «а ты умеешь жарить блины? Мама даёт тебе муку и яйца, ты — блины. Это и есть функция».
Теория · синтаксис

Анатомия def

first_function.py
Python
def greet(name): return f"Привет, {name}!" msg = greet("Маша") print(msg)

def greet(name):

def — объявление, greet — имя, (name) — параметр, : — начинаем «тело».

return f"..."

Возвращаем результат «наружу». Кто вызвал — тот и получает строку.

greet("Маша")

Вызов функции. Передаём аргумент «Маша» — он попадёт в параметр name.

Если ученик забыл return — функция вернёт None. Скажи: «Если нужен результат наружу — return. Если только действие (print, f.write) — не нужно».
Частая путаница

Параметр vs Аргумент

Это два разных слова, которые обозначают почти одно и то же — но разницу важно понимать.

Параметр

На вывеске рецепта

Имя, которое мы пишем в def. Это переменная внутри функции — мы ещё не знаем, что туда попадёт.

def bake(flour):
    #       ↑
    #   параметр
    ...
Аргумент

Реальная мука, которую сыплешь

Значение, которое мы передаём при вызове. Конкретное число, строка, список.

bake("гречневая мука")
#      ↑
#   аргумент
Мнемоника: Параметр — на Платформе-вывеске. Аргумент — Активный, живой. Если ребёнок путает — не страшно, программисты тоже путают.
Ключевое различие

return или print?

print — показать

Напечатал — и всё. Забыл. Ничего не вернулось.

Python
def square(x): print(x * x) # показал r = square(5) # r == None print(r + 10) # TypeError!

return — отдать

Вернул — кто вызвал, тот получил. Можно дальше считать.

Python
def square(x): return x * x # отдал r = square(5) # r == 25 print(r + 10) # 35

Правило: если результат нужен дальше в коде — return. Если только показать пользователю — можно print.

Это одна из главных идей урока. Обязательно запусти оба варианта на экране и покажи ошибку TypeError: unsupported operand type(s) for +: 'NoneType' and 'int'.

Step 1 / 3 Задание · 8 минут

greet(name) — первая функция с параметром

Напиши функцию, которая получает имя и возвращает приветствие (не печатает!). Вызови трижды с разными именами.

greet.py · шаблон
Python
def greet(name): return f"Привет, {name}!" print(greet("Дима")) print(greet("Катя")) print(greet("Ваня"))

Что должно получиться

Привет, Дима!
Привет, Катя!
Привет, Ваня!
Застрял? Нажми на подсказку

Забыл return? Тогда greet("Дима") вернёт None, а print(None) напечатает «None». Вставь return перед f-строкой.

Пишет SyntaxError? Проверь двоеточие в конце def greet(name): и отступ в 4 пробела перед return.

Следи: ученик должен использовать именно return, а не print внутри функции. Если пишет def greet(name): print(...) — похвали за вариант, но объясни: так мы не сможем сохранить результат в переменную.
Step 2 / 3 Задание · 10 минут

calc_age(birth_year) — функция с расчётом

Теперь функция не просто форматирует строку, а делает арифметику. Возвращает число.

calc_age.py
Python
def calc_age(birth_year): return 2026 - birth_year # Вызови для разных лет print(calc_age(2014)) # 12 print(calc_age(2010)) # 16 print(calc_age(1990)) # 36 # Можно сохранить и использовать my_age = calc_age(2013) print(f"Мне {my_age} лет, в днях — {my_age * 365}")

Фишка этого урока

Результат calc_age(2013) — это обычное число. С ним можно считать, сравнивать, передавать в другие функции.

Бонус-задание

Добавь вторую функцию:

def age_in_days(age):
    return age * 365

Теперь можно: age_in_days(calc_age(2013))

Это важный момент: показать, что функцию можно вложить в другую функцию. age_in_days(calc_age(2013)) — сначала считается calc_age, потом age_in_days. Как матрёшка.

Вот как выглядит твой diary.py сейчас

diary.py · до рефакторинга
Python
from datetime import datetime while True: cmd = input("1 — написать, 2 — показать, 3 — выйти: ") if cmd == "1": text = input("Что нового? ") now = datetime.now().strftime("%d.%m.%Y %H:%M") with open("diary.txt", "a", encoding="utf-8") as f: f.write(f"[{now}] {text}\n") elif cmd == "2": with open("diary.txt", "r", encoding="utf-8") as f: print(f.read()) elif cmd == "3": break

Step 3 / 3 Рефакторинг · 15 минут

А вот — то же самое, но чисто

diary_v2.py · после рефакторинга
Python
from datetime import datetime def write_entry(text): now = datetime.now().strftime("%d.%m.%Y %H:%M") with open("diary.txt", "a", encoding="utf-8") as f: f.write(f"[{now}] {text}\n") def read_all(): with open("diary.txt", "r", encoding="utf-8") as f: return f.read() def count_entries(): with open("diary.txt", "r", encoding="utf-8") as f: return len(f.readlines()) while True: cmd = input("1 — написать, 2 — показать, 3 — сколько, 4 — выйти: ") if cmd == "1": write_entry(input("Что нового? ")) elif cmd == "2": print(read_all()) elif cmd == "3": print(f"Записей: {count_entries()}") elif cmd == "4": break
Главная идея: мы не добавили ни одной новой функциональности (кроме count_entries), но код стал намного понятнее. Главный цикл читается как оглавление: «написать», «показать», «сколько», «выйти».
Архитектура после рефакторинга

Три функции — три ответственности

Главный цикл — дирижёр. Функции — музыканты. Каждый играет свою партию.

Подчеркни: функция read_all() — без параметров, но с return. write_entry(text) — с параметром, но без return (ничего не возвращает наружу). Это показывает: комбинации разные, но принцип один.

Если что-то пошло не так

5 типичных граблей

TypeError: NoneType

Забыл return. Функция ничего не вернула. Добавь return перед значением.

NameError

Вызываешь функцию до её определения. Python читает сверху вниз: сначала def, потом вызов.

Функция без ()

greet — это указатель. greet() — это вызов. Без скобок функция не запустится.

Локальные vs глобальные

Переменная внутри def — «своя». Снаружи её не видно. Пока: не используй одинаковые имена внутри и снаружи.

Бонус-ошибка: параметр vs аргумент

«Параметр — на вывеске рецепта, аргумент — реальная мука, которую сыплешь». Повтори 3 раза — запомнится.

Не заставляй запоминать всё сразу. Достаточно, чтобы ребёнок узнавал «о, это та самая ошибка, про неё говорили» — остальное найдёт сам.

Задача
