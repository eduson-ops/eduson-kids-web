Теория · Метафора

Переменная — это коробка с наклейкой

На наклейке — имя. Внутри — значение. Хочешь — меняй содержимое, наклейка остаётся. Как ящик в комнате с биркой «носки»: что внутри — твоё дело, но по бирке ты его находишь.

name "Дима"
age 11
city "Москва"

name = "Дима"
age = 11
print(name, age)

Теория · Типы данных

У Python есть разные коробки — для разного

"11" — это строка (буквы). 11 — это число. С числами можно считать, со строками — нет.

strСтрока
"Привет" · "11" · "Дима"
Любой текст в кавычках. Кавычки любые, главное — парные.
intЦелое число
11 · 2026 · -7 · 365
Целые числа без точки. С ними можно делать математику.
floatДробное
3.14 · 1.5 · 0.25
Числа с точкой. Пригодятся для денег, процентов, скорости.

Важно   "11" + "11" = "1111" (склеиваем строки), а 11 + 11 = 22 (математика). Один и тот же знак + работает по-разному — всё зависит от типа.

Главный секрет   input() всегда возвращает строку — даже если ты набрал число. На следующем слайде увидишь, почему это важно.

Ошибка, которая научит

Почему нельзя просто 2026 - input(...)?

Потому что input() отдаёт строку. А из числа вычесть строку Python не умеет. Сейчас увидим ошибку своими глазами — и научимся её чинить.

почти-работает.py
Python
birth = input("Год рождения? ") age = 2026 - birth # а тут БУМ print(age)
ошибка
$ python age.py
Год рождения? 2014
TypeError: unsupported operand
type(s) for -: 'int' and 'str'

Прочитай сообщение медленно. Какие слова повторяются? int и str. Значит Python запутался в типах — надо превратить строку в число через int(...).

Практика · Задание 3 · 12 минут

Калькулятор возраста

Спрашиваешь год рождения → превращаешь в число через int(...) → считаешь возраст → выводишь красиво через f-строку.

age_calc.py
Python
name = input("Как тебя зовут? ") birth_year = int(input("В каком году ты родился? ")) age = 2026 - birth_year print(f"Привет, {name}!") print(f"Тебе {age} лет.") print(f"А в днях — {age * 365} дней.")
работает!
$ python age_calc.py
Как тебя зовут? Дима
В каком году ты родился? 2014
Привет, Дима!
Тебе 12 лет.
А в днях — 4380 дней.
Приём профи

f-строки — вставляем переменные прямо в текст

Буква f перед кавычками говорит Python: «внутри могут быть переменные в фигурных скобках». Python их подставит.

ПО-СТАРОМУ

Через запятую

print("Тебе", age, "лет")

Работает, но выглядит неаккуратно. Пробелы появляются сами — не всегда удобно.

ПО-НОВОМУ

Через f-строку

print(f"Тебе {age} лет, а в минутах — {age * 365 * 24 * 60}")

Внутри фигурных скобок можно писать любые выражения — даже умножение.

Демо и рефлексия · 55–60 мин

Два вопроса на подумать

ВОПРОС 1

Что было бы, если написать 2026 - input(...) без int?

Python не умеет вычитать строку из числа. Выдал бы TypeError — «не могу вычесть str из int». input() всегда отдаёт строку, её надо кастить.

ВОПРОС 2

Зачем нужна f-строка?

Чтобы красиво вставлять переменные в текст. Без неё пришлось бы через запятые и пробелы — получается грязно. С f — как родной русский.

«Ты сам догадался про int() после первой ошибки — это уровень, не все взрослые сразу понимают».

Что может пойти не так

Типичные сложности и как их починить

Забыл int(). Помни: ввод — всегда строка.
