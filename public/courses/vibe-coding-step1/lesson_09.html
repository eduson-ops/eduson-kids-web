ЦЕЛИ УРОКА

Что ты будешь уметь через 60 минут

Знать

Что такое API-токен, зачем нужен файл .env, библиотека python-dotenv, OAuth, Bearer-токен.

Понимать

Что API-ключи — это пароли. Их нельзя показывать в коде, выкладывать в чат или публиковать на GitHub.

Уметь

Сделать POST-запрос к GigaChat с авторизацией: сначала получить access-токен, потом отправить вопрос и прочитать ответ.

Владеть

Минимальным системным промптом — роль ИИ, тон ответа, что ему можно и нельзя. Задашь «характер» — получишь нужный ответ.

API-ключ — это пароль от чужого сервера

Когда ты используешь чужой сервис (GigaChat, ЦБ РФ, погода) через код — сервер должен понять, кто ты такой. Для этого ему предъявляют ключ.

ПОЛУЧАЕШЬответ нейросети
Правило №1 вайб-кодера.

Если потерял ключ — любой может зайти и потратить твои лимиты. Сгенерируй новый в Studio и сразу удали старый. Публично показывать ключ — нельзя, даже преподавателю в зуме.

ТЕОРИЯ · 2 из 3

Файл .env — тайник для паролей

ПЛОХО Ключ прямо в коде

  • Загружаешь на GitHub — ключ виден всему миру
  • Скинул папку в чат — друг может взять
  • При каждой замене ключа правишь код
  • Боты сканируют GitHub и крадут ключи за часы

ХОРОШО Ключ в .env

  • Код в репозитории, а .env — только у тебя
  • .gitignore не даёт случайно загрузить
  • Смена ключа — правишь одну строку в одном файле
  • Так делают все настоящие разработчики
Библиотека python-dotenv читает файл .env и кладёт значения в переменные окружения. В коде берёшь через os.getenv("GIGACHAT_CREDENTIALS").

СЛОВАРЬ УРОКА

7 терминов, которые войдут в лексикон

API-ключ

Строка-пароль, по которой сервис узнаёт тебя. Не отдавать никому.

Client ID

Твой публичный идентификатор — «кто я». Можно показывать.

Client Secret

Секретная вторая половина ключа. СЕКРЕТ

Bearer-токен

Временный пропуск в заголовке Authorization: Bearer ...

.env

Файл в корне проекта, где лежат пароли. Игнорируется в git.

POST-запрос

Отправка данных на сервер (в отличие от GET — просто получение).

Системный промпт SYSTEM PROMPT

Настройка «характера» нейросети в первом сообщении: «Ты — умный друг 11-летнего ребёнка. Отвечай коротко и с юмором.» Это твой способ управлять ИИ.

ШАГ 4 / 6 · ПИШЕМ КОД

first_ai.py — скелет запроса

first_ai.py · Python 3.11
Python
import os import uuid import requests from dotenv import load_dotenv load_dotenv() # читаем .env в переменные AUTH_KEY = os.getenv("GIGACHAT_CREDENTIALS") # достаём наш ключ # ШАГ 1. Получаем access_token на 30 минут oauth_url = "https://ngw.devices.sberbank.ru/api/v2/oauth" token_resp = requests.post( oauth_url, headers={ "Authorization": f"Basic {AUTH_KEY}", "RqUID": str(uuid.uuid4()), # уникальный ID запроса "Content-Type": "application/x-www-form-urlencoded", }, data={"scope": "GIGACHAT_API_PERS"}, verify=False, # обход сертификата ) access_token = token_resp.json()["access_token"] print("Токен получен:", access_token[:20], "...") # печатаем первые 20 символов
Подсказка: что делает uuid.uuid4()

Это генерация случайного уникального ID — как серийный номер у билета. Нужно серверу, чтобы различать одинаковые запросы. Мы про это ещё поговорим, а сейчас просто используем.

ТИПОВЫЕ СЛОЖНОСТИ

Если не сработало — смотри сюда

401 Unauthorized

Симптом: сервер не пускает.
Лечение: ключ неправильный или истёк — сгенерируй новый в Studio, обнови в .env.

SSL certificate_verify_failed

Симптом: Python не доверяет серверу Сбера.
Лечение: поставь verify=False в обоих requests.post (временный костыль).

ModuleNotFound 'dotenv'

Симптом: Python не знает dotenv.
Лечение: в терминале pip install python-dotenv. Имя пакета с дефисом.

None вместо ключа

Симптом: AUTH_KEY пустой.
Лечение: проверь, что .env лежит в той же папке, что и first_ai.py. Точно с точкой спереди.
