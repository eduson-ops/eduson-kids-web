Теория-демо · шаг 1

Записываем в файл первые байты

Три вещи в одной строке: что открыть, в каком режиме, с какой кодировкой.

  • "test.txt" — имя файла (создастся, если нет)
  • "w" — режим «записать»
  • encoding="utf-8" — понимает кириллицу
  • with — автозакрытие в конце блока
Открой Проводник, найди созданный файл — это реальный .txt на твоём диске.
first_write.py
Python
with open("test.txt", "w", encoding="utf-8") as f: f.write("Привет из Python!") # Python откроет файл, # запишет строку, # и сам закроет. # Ничего не нужно делать руками.
Заметки преподавателя

После запуска ребёнок должен открыть файл в Проводнике глазами — это магия момента. Текст, написанный в коде, лежит на диске реальным файлом.

Можешь открыть в Блокноте — покажет ту же строку. Это подтверждение: «файл — не что-то особенное, это обычный текст».

Ключевая механика

Три режима открытия файла

"r"

read — читать

Только смотрим содержимое. Ничего не изменится. Если файла нет — ошибка.

"w"

write — переписать

Всё старое стирается. Пишем заново с нуля. Осторожно — работы можно потерять.

"a"

append — дописать

Старое остаётся, новое добавляется в конец. Для дневника — идеально.

Правило для дневника: всегда "a". Каждая новая запись — дописывается, ничего не трётся.
Заметки преподавателя

Это ключевой слайд урока. Проверяй, что ребёнок понимает разницу — один из критериев успешного занятия.

Реальная история: Ребята часто случайно затирают свои записи, открыв в «w» вместо «a». Пусть один раз это случится сознательно — откроет в «w», увидит, что всё пропало, поймёт страх и запомнит навсегда.

Теория-демо · шаг 2

Читаем файл обратно

Тот же with open, только режим — "r", а вместо .write() используем .read().

  • f.read() — всё содержимое одной строкой
  • f.readlines() — список строк
  • Цикл for line in f — построчно, экономно
read_file.py
Python
with open("my_file.txt", "r", encoding="utf-8") as f: content = f.read() print(content) # или построчно: with open("my_file.txt", "r", encoding="utf-8") as f: for line in f: print(line.strip()) # ↑ убирает лишний \n
Заметки преподавателя

Объясни .strip(): «в конце каждой строки есть невидимый переход на новую строку \n — он и так есть в print, получается двойной отступ. .strip() его съедает».

На этом этапе — задание 2 в ЛП (10 минут). Ребёнок читает файл, видит то же, что записал ранее.

Теория-демо · шаг 3

Дата и время из модуля datetime

Чтобы дневник был дневником, каждой записи нужна метка времени. За это отвечает стандартный модуль Python.

  • datetime.now() — текущая дата и время
  • .strftime(...) — форматирование в строку
  • %d.%m.%Y — день.месяц.год
  • %H:%M — часы:минуты
Лайфхак: модуль и класс называются одинаково — datetime. Не пугайся двойного импорта.
time_stamp.py
Python
from datetime import datetime now = datetime.now() print(now) # 2026-04-20 14:33:21.837194 stamp = now.strftime("%d.%m.%Y %H:%M") print(stamp) # 20.04.2026 14:33 # Готовая метка для дневника.
Заметки преподавателя

Здесь у детей часто ступор: «from datetime import datetime» — модуль называется как класс. Объясни сразу: «папка и файл внутри с одним именем».

Формат %d.%m.%Y не обязательно запоминать — можно подсматривать в памятке. Это нормально.

Собираем дневник по шагам

Типичные ошибки

Не пугайся, вот решения

!

UnicodeDecodeError

Забыл encoding="utf-8". Python без него не понимает русский. Добавь везде — и в read, и в write.

!

Файл не находится

Запускай python diary.py из папки, где лежит сам файл. Проверь cd в терминале.

!

Файл пропал после запуска

Открыл в "w" и затёр. Всегда "a", если не хочешь перезаписывать.

!

datetime не работает

Нужно from datetime import datetime. Модуль и класс с одним именем — это нормально.

!

Кракозябры в файле

Опять UTF-8. Возможно, открыл файл в старом Блокноте — попробуй VS Code.

!

f.write(input) вместо input()

Без скобок input — это функция-объект. Со скобками — вызов. Запомни: скобки = действие.

Заметки преподавателя

Ошибки — это нормально. Фраза для ребёнка: «Ошибка — это не плохо, это просто письмо от компьютера». Читай вместе верхнюю красную строку в терминале.

Подкрепление: encoding="utf-8" — пиши сразу, даже не задумываясь. Это must-have.

Домашнее задание · до 15 минут

Функция «поиск по слову»

Добавь в дневник команду «найти»: спрашивает у пользователя слово, читает файл построчно, выводит только те строки, в которых это слово встречается.

Подсказки:

  • Открываешь diary.txt в режиме "r".
  • for line in f: — перебирай построчно.
  • if слово in line: — проверка вхождения.
  • Если нашёл — print(line).
search_example.py
Python
word = input("Какое слово ищем? ") with open("diary.txt", "r", encoding="utf-8") as f: found = False for line in f: if word.lower() in line.lower(): print(line.strip()) found = True if not found: print(f"Слов '{word}' не найдено.")
Формат сдачи: файл diary.py + скриншот с поиском в TG-чат преподавателю.

Бонус для опережающих

Дневник с фильтром по дате

Если ты уже управился раньше времени — задача со звёздочкой. Нужно вывести только те записи, которые были сделаны за последние 7 дней.

  • Парсим дату из начала строки: datetime.strptime(...)
  • Сравниваем с datetime.now() - timedelta(days=7)
  • Выводим только свежие
bonus_filter.py
Python
from datetime import datetime, timedelta week_ago = datetime.now() - timedelta(days=7) with open("diary.txt", "r", encoding="utf-8") as f: for line in f: # строка вида "[20.04.2026 14:33] текст" date_str = line[1:17] # вытащили "20.04.2026 14:33" entry_date = datetime.strptime(date_str, "%d.%m.%Y %H:%M") if entry_date >= week_ago: print(line.strip())
Заметки преподавателя

Это уровень 6–7 уроков. Обычный ребёнок сюда не дойдёт — и это нормально. Бонус для тех, кто «заскучал» и хочет больше.

Сложность: strptime (парсинг) против strftime (форматирование). Часто путают. Объясни: «p» — parse, «f» — format.
