Зачем нам это

Вчера функция умела одно

Функция greet() всегда печатала одно и то же: «Привет!». А что если надо поздороваться с Аней, Борей и Витей — по-разному?

Писать три функции? Нет. Напишем одну — и научим её принимать параметры.

before.py · было
Python
def greet(): print("Привет!") greet() # Привет! greet() # Привет! greet() # Привет! (скучно)
Что получится

Стало: одна функция — много имён

after.py · стало
Python
def greet(name): print(f"Привет, {name}!") greet("Аня") greet("Боря") greet("Витя")
output · консоль 
Привет, Аня!
Привет, Боря!
Привет, Витя!
ℹ
Магия в одной строке
Добавили name в скобки при def. Внутри подставили через f"{name}". Всё! Функция теперь умеет работать с любым именем.

Главное понятие

Параметр vs аргумент

📮
параметр

имя переменной в def

Стоит в скобках при объявлении функции.

←
✉️
аргумент

значение при вызове

То, что мы реально передаём внутрь.

разбор.py
Python
def greet(name): # name — ПАРАМЕТР (почтовый ящик) print(f"Привет, {name}!") greet("Аня") # "Аня" — АРГУМЕНТ (письмо) greet("Витя") # "Витя" — тоже АРГУМЕНТ
Несколько параметров

2, 3, хоть 10 — через запятую

add.py
Python
def add(a, b): print(a + b) add(3, 5) # 8 add(10, 7) # 17 add(1, 1) # 2
rectangle.py
Python
def print_rectangle(w, h, sym): for _ in range(h): print(sym * w) print_rectangle(5, 3, "#") print_rectangle(8, 2, "*") print_rectangle(3, 4, "@")

Параметры в def — имена через запятую. Аргументы в вызове — значения через запятую. В том же порядке.

Порядок решает

Поменял порядок — сломал вывод

Python не читает смысл слов. Он раскладывает аргументы по параметрам в том порядке, в каком ты их передал.

Первый аргумент → первый параметр. Второй → второй. И так далее.

⚠️
Частая ошибка
Дети пишут greet("Боря", "Аня"), думая «ну это же оба имена». Python думает иначе.
порядок.py
Python
# правильно: w=5, h=3, sym="#" print_rectangle(5, 3, "#") # ##### # ##### # ##### # неправильно: w=3, h="#", sym=5 print_rectangle(3, "#", 5) # TypeError: range() expected int # (Python пытается range("#") — бред)
Практика 1 · 8 минут

Функция print_rectangle

  • Три параметра: width, height, symbol.
  • Внутри: цикл for печатает symbol * width, height раз.
  • Вызов: минимум 3 раза с разными значениями.
rectangle.py
Python
def print_rectangle(width, height, symbol): for _ in range(height): print(symbol * width) print_rectangle(6, 3, "#") print_rectangle(10, 2, "*") print_rectangle(4, 4, "@")

Практика 3 · 10 минут

Убираем глобальную переменную

было.py
Python
maze = [ "#####", "#...#", "#####", ] def show_map(): for row in maze: # тянет глобальную print(row) show_map()
стало.py
Python
maze1 = ["#####", "#...#", "#####"] maze2 = ["###", "#.#", "#.#", "###"] def show_map(maze): # параметр! for row in maze: print(row) show_map(maze1) show_map(maze2) # та же функция!
💎
Главная идея
Функция получает данные через параметр, а не тянет их «снаружи». Теперь show_map работает с любой картой.

Параметры + циклы и условия

Python
def say_hi(name, times): for _ in range(times): print(f"Привет, {name}!") say_hi("Аня", 3) # Привет, Аня! # Привет, Аня! # Привет, Аня!
условие.py
Python
def check_age(name, age): if age >= 18: print(f"{name}, взрослый") else: print(f"{name}, ещё маленький") check_age("Витя", 12) check_age("Мама", 40)

Один параметр — ключ к сотне сценариев. Сочетай смело.

Важный нюанс

Параметр живёт только внутри

Параметр — это внутренняя переменная функции. Снаружи её нет.

Когда функция завершилась — «коробка» с параметром выбрасывается. Как временный столик в кафе после обеда.

scope.py
Python
def greet(name): print(f"Привет, {name}!") greet("Аня") # ок print(name) # NameError! # name живёт только # внутри функции
Грабли

Топ ошибок и как чинить

🧩 Путают параметр и аргумент

Параметр — почтовый ящик (имя в def). Аргумент — письмо (значение при вызове). Повторяй аналогию.

⚠️ TypeError: missing argument

Забыли передать. Читаем ошибку вместе: Python пишет, каких именно аргументов не хватает.

🔀 Неправильный порядок

print_rectangle(3, "#", 5) — Python не понимает смысл, просто расставляет по порядку.

📦 Меняют параметр — ждут, что снаружи тоже

Параметр живёт только внутри функции. Снаружи — старое значение. Это отдельная коробка.

def draw_square(size, fill, border): # верхняя граница print(border * size) # средние ряды: border + fill * (size-2) + border for _ in range(size - 2): print(border + fill * (size - 2) + border) # нижняя граница print(border * size) print() # пустая строка для красоты # пять вызовов — пять разных артов draw_square(5, ".", "#") draw_square(7, "*", "@") draw_square(4, " ", "X") draw_square(6, "o", "=") draw_square(8, "~", "+")
