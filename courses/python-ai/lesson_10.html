Зачем нам логические операторы

Программа умеет проверять одно условие. Но жизнь — сложнее.

Вчера

«Если возраст ≥ 10 — ты школьник».
Но что, если 100 лет?

Сегодня

«Если возраст от 10 до 18 — школьник». Два условия сразу.

Завтра

Банкомат, турникет, защита пароля — всё это связки условий.

Сегодня освоим 3 слова, которые превращают код из ребёнка в программу-профи: and, or, not.

Цель и результат

  • что такое логические операторы и зачем они нужны
  • назначение and, or, not
  • как объединять несколько проверок в одну
  • что результат всегда True или False

Умеешь

  • использовать логические операторы в if
  • писать программы с диапазонами
  • читать чужие сложные условия

Три слова, три метафоры

Запомни короткой фразой

оператор
and
«и то, и то»

Оба условия должны быть True. Строгий.
оператор
or
«хоть что-то»

Хотя бы одно True — и уже True. Щедрый.
оператор
not
«наоборот»

Переворачивает True ↔ False.

Эти три слова — готовые кнопки Python. Пишешь прямо в условии.

Оператор and

and — строгий охранник

Пропустит, только если всё правда.

«Пойду гулять, если на улице солнце И нет дождя».

Достаточно одной лжи — и весь результат становится False.

ABA and B
TrueTrueTrue
TrueFalseFalse
FalseTrueFalse
FalseFalseFalse
Python
age = 15 if age >= 10 and age <= 18: print("Школьник")
Оператор or

or — щедрый друг

Дал шанс — даже если только одно правда.

«Получу мороженое, если хорошо себя вёл ИЛИ помыл посуду».

Оба ложны — только тогда False.

ABA or B
TrueTrueTrue
TrueFalseTrue
FalseTrueTrue
FalseFalseFalse
Python
age = 5 if age < 7 or age > 80: print("Бесплатный билет")
Оператор not

not — перевёртыш

Берёт True — делает False. И наоборот.

True → False

False → True

Удобен для вежливых проверок: «если НЕ совершеннолетний — спрашиваем родителей».

Python
age = 15 if not (age == 18): print("Не совсем взрослый") # ещё пример is_night = False if not is_night: print("Доброе утро!")

Скобки делают not понятнее: not (a == 18) читается лучше, чем not a == 18.

Приём профи

Python умеет писать диапазон коротко

Можно так (длинно)

Python
if age >= 10 and age <= 18: print("Подросток")

А можно так (элегантно)

Python
if 10 <= age <= 18: print("Подросток")

Двойное неравенство — фишка Python. Читается как на уроке математики.

Теория-демо · 5–15 мин

Разбираем logic_demo.py

age = 15

# AND — строгий: оба должны быть True
if age >= 10 and age <= 18:
    print("Школьник")

# OR — щедрый: хотя бы одно True
if age < 7 or age > 80:
    print("Бесплатный билет")

# NOT — переворачивает результат
if not (age == 18):
    print("Не совсем взрослый")

Запусти. Меняй age на 5, 18, 99 — смотри, какие ветки срабатывают.

Практика · 8 мин

Задача 1. «Подросток»

Запросить возраст. Вывести «Подросток», если 10 ≤ возраст ≤ 17.

План

  • прочитать число через input() и int()
  • проверить диапазон через and
  • если попали — печатаем «Подросток»
  • иначе — «Ещё не подросток / уже взрослый»
Python
age = int(input("Возраст: ")) if 10 <= age <= 17: print("Подросток") else: print("Не подросток")

Мини-проект урока · 45–55 мин

Банкомат-проверка 🏧

Первый настоящий сценарий с составным условием.

Что делает программа

  • спрашивает пин-код
  • спрашивает сумму запроса
  • хранит баланс (например, 5000)
  • выдаёт деньги только если:
    пин = «1234» И сумма ≤ баланса
  • иначе — объясняет, что не так
Python
balance = 5000 pin = input("ПИН: ") sum_ = int(input("Сумма: ")) if pin == "1234" and sum_ <= balance: print("Выдано ✅") elif pin != "1234": print("Неверный ПИН") else: print("Недостаточно денег")
Типичные ошибки

5 засад, в которые попадают все

1. Сокращённое сравнение

age > 10 and < 20 — SyntaxError.
Надо: age > 10 and age < 20.

2. Путают and и or

Нарисуй таблицу истинности. Тренируйся словами: «и» / «или».

3. Порядок операций

and приоритетнее or. Ставь скобки для ясности.

4. not с числами

not 0 = True, not 5 = False. Пока используем not только с True/False.

5. Пробел внутри оператора

a nd b — тоже SyntaxError. Пиши слитно: and, or, not.

Важное правило

Порядок важен: and сильнее, чем or

Без скобок

Python
# Python читает как: # a or (b and c) a = True b = False c = False print(a or b and c) # True

Со скобками

Python
# Смысл совсем другой a = True b = False c = False print((a or b) and c) # False

Совет: если есть и and, и or — ставь скобки. Читатель скажет спасибо (и ты сам через неделю).

Рефлексия · 55–60 мин

Быстрый мозговой штурм

True and False = ?
→ False (одно ложно — всё ложно)
True or False = ?
→ True (хоть один True)
not (5 > 2) = ?
→ False (5>2 = True, not True = False)
10 <= 15 <= 18 = ?
→ True (15 между 10 и 18)
Домашнее задание · ≤ 20 мин

Программа «Регистрация» 🛡️

Приложение принимает новых пользователей — но только по трём правилам сразу.

Три правила (все через and)

  • логин длиннее 3 символов
  • пароль длиннее 5 символов
  • логин ≠ паролю
Python
login = input("Логин: ") password = input("Пароль: ") if (len(login) > 3 and len(password) > 5 and login != password): print("Регистрация успешна ✅") else: print("Проверь правила ❌")
Бонус для опережающих 🚀

Високосный год

Год високосный, если он делится на 4 И не делится на 100 ИЛИ делится на 400.

year = int(input("Год: "))

if (year % 4 == 0 and year % 100 != 0) or year % 400 == 0:
    print(f"{year} — високосный 🎉")
else:
    print(f"{year} — обычный")

Проверь: 2024 → високосный. 2100 → нет (делится на 100, но не на 400). 2000 → да.
