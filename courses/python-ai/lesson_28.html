Архитектура · сердце урока

Три структуры — три роли

кортеж

WORDS

WORDS = ("python", "кортеж", ...)

Неизменяемый список слов. Во время игры слова не должны меняться — поэтому кортеж, а не список.

множество

guessed

guessed = set()

Угаданные буквы. Дубликаты запрещены — одну букву нельзя засчитать дважды. Множество — это гарантия.

словарь

stats

stats = {"победы": 0, "поражения": 0}

Статистика по всем играм. Ключ → число. Легко обновлять: stats["победы"] += 1.

💡
Главная мысль для ребёнка
«Правильная структура данных = половина программы. Вторая половина — логика.»

Функция №1 · 15–20 мин

Функция show

Задача: превратить "кошка" + угаданные {"к", "а"} в красивую строку к _ _ к а.

show(word, guessed)
  • Проходим по каждой букве слова
  • Если буква в множестве — показываем её
  • Если нет — вместо неё _
  • Соединяем через пробел: " ".join(...)
«Сначала проговори вслух, что возвращает функция. Потом пиши.» — ключевая фраза преподавателя
show.py
Python
def show(word, guessed): return " ".join( c if c in guessed else "_" for c in word ) # проверяем: print(show("кошка", {"к", "а"})) # → к _ _ к а
Случайность · 20–25 мин

Выбираем слово случайно

random.choice — любимая функция недели

Берёт один случайный элемент из любой коллекции: кортежа, списка, даже строки.

  • Импорт один раз в начале файла
  • Работает и на одном элементе — не боимся
  • Каждый запуск — новое слово
  • Это и есть «живость» игры
choose.py
Python
import random WORDS = ("python", "кортеж", "множество", "словарь", "виселица") secret = random.choice(WORDS) print(secret) # "словарь" (или любое другое)
Самостоятельно · 25–45 мин

Главный while-цикл

Ребёнок пишет сам. Преподаватель — только наводящими вопросами.

  1. Начать с lives = 6 и guessed = set()
  2. Внутри цикла — показать слово и жизни
  3. Считать букву, привести к нижнему регистру
  4. Проверить — не была ли уже
  5. Добавить в множество, сбавить жизни если ошибка
  6. Выйти когда слово угадано или жизни кончились
main_loop.py
Python
lives = 6 guessed = set() while lives > 0: shown = show(secret, guessed) print(shown, f"жизни: {lives}") if "_" not in shown: print("Победа!", secret) break letter = input("Буква: ").lower().strip() if letter in guessed: print("Уже было"); continue guessed.add(letter) if letter not in secret: lives -= 1 else: print("Поражение:", secret)

hangman.py · ~30 строк
Python
import random WORDS = ("python", "кортеж", "множество", "словарь", "виселица") def play(): secret = random.choice(WORDS) guessed = set() lives = 6 while lives > 0: shown = " ".join(c if c in guessed else "_" for c in secret) print(shown, f"жизни: {lives}") if "_" not in shown: print("Победа! Слово было:", secret); return True letter = input("Буква: ").lower().strip() if letter in guessed: print("Уже было"); continue guessed.add(letter) if letter not in secret: lives -= 1 print("Поражение. Слово было:", secret) return False stats = {"победы": 0, "поражения": 0} while True: if play(): stats["победы"] += 1 else: stats["поражения"] += 1 print(stats) if input("Ещё? д/н: ") != "д": break

Типичные ловушки

Что пойдёт не так — и как чинить

⚠️
Забыл .lower()
Ввёл «А», слово — «а». Лишняя жизнь в минус. Лечится одной цепочкой: .lower().strip().
⚠️
Не учитывает повтор буквы
Тратит жизни зря на одну и ту же. Чиним: if letter in guessed: continue.
⚠️
Боится random.choice на одном элементе
Работает штатно. Успокаиваем — покажи и запусти прямо на одном слове.
⚠️
Путает множество и список
Напоминаем: дубликаты нам не нужны — значит множество.
⚠️
Бесконечный цикл без break
Ctrl + C — спокойно останавливаем. Разбираем, какое условие забыто.
