Проверяем падение яблока

Что должно быть
  • Фон, корзинка, облако — на месте с урока 37
  • В images/ появилось apple.png (Kandinsky, 200×200, прозрачный)

Типичные баги
  • Яблоко «стоит» — забыт += или строка в draw(), а не в update()
  • Яблоко пропало — улетело ниже 600, это нормально, фикс — на уроке 39
  • No such file — имя с большой буквы: Apple.png ≠ apple.png

Показываю свой пример, затем собираем у каждого 20-секундный скрин геймплея — оценка «падает ли».

Держи этот блок строго 5 минут. Длинный разбор ДЗ выжигает энергию перед новой темой. Если у одного-двух детей яблоко не падает — фиксируй в заметках, починишь на блоке 15-30 в их собственном коде.
Главная идея урока

Событие и состояние — это разные вещи

Event · вспышка

Событие

Произошло одноразово и пропало. Кликнул — один раз сработало.

  • on_mouse_down(pos, button)
  • on_mouse_up(pos)
  • Метафора: звонок в дверь
State · проверка

Состояние

Всегда можно спросить: «сейчас — да или нет?». Например, мышь зажата.

  • on_mouse_move(pos) — поток апдейтов
  • Похоже на keyboard.left (будет на уроке 42)
  • Метафора: держишь рукоятку

Правило: прыжок — событие, бег — состояние. Сегодня тренируемся на мыши, с клавиатурой применим тот же приём через 4 урока.

Это стержневая идея всего блока PyGame Zero. Объясни через аналогию звонка в дверь: никто же не стоит у двери и не спрашивает «а нажимают ли звонок сейчас?» — дверь сама сообщает. Запиши на доске/бумаге два столбца. Уточни у ребёнка: «приход смс — событие или состояние?» (событие). «Заряд батареи — событие или состояние?» (состояние).
5-15 мин · теория

Три обработчика мыши в PyGame Zero

Курсор движется

on_mouse_move(pos)

Вызывается много раз, пока мышь двигается. pos — кортеж (x, y).

Кнопку нажали

on_mouse_down(pos, button)

Одна вспышка. button: mouse.LEFT, RIGHT, MIDDLE.

Кнопку отпустили

on_mouse_up(pos)

Тоже одна вспышка. Пригодится для «оттащить и бросить».

Живая демка — пишу прямо сейчас
Python
def on_mouse_down(pos): print(pos) # кликаю — в консоли выскакивает (x, y)

Запускаю игру, кликаю в 5 точек, показываю, как появляются пары чисел. Это доказательство, что PyGame Zero действительно вызывает функцию.

Главная мысль слайда: «PyGame Zero сам ищет функции с именами on_...». Объясни, что для IDE это просто обычные функции — никакой магии. Обязательно покажи живую консоль с print(pos). Для ребёнка это «ого, действительно работает».
Мини-напоминание

Система координат: (0, 0) — верхний левый

ось X — вправо, ось Y — вниз (не как в математике)

pos — это кортеж

Такой же, как списки из урока 6, только не меняется.

  • pos[0] — координата x
  • pos[1] — координата y
  • Можно распаковать: x, y = pos
Y растёт вниз. Это главное «сбивает с толку» в играх. Привыкнешь за один урок.
Рисуем сеточку в стиле тетради, но (0,0) — слева сверху. Спроси: «если корзинка в (400, 520), где она?» (снизу по центру). Это опорная точка для всего сегодняшнего кода.
15-30 мин · совместный кодинг · шаг 1

Корзинка следит за курсором

Python
import pgzrun WIDTH, HEIGHT = 800, 600 basket = Actor('basket', (400, 520)) apple = Actor('apple', (400, 50)) def draw(): screen.fill('skyblue') basket.draw() apple.draw() def update(): apple.y += 2 def on_mouse_move(pos): basket.x = pos[0] # pos[0] — это x pgzrun.go()
Что происходит построчно
1
Импорт и размер окна — базовое, как на уроке 37.
2
Два Actor — корзинка внизу, яблоко сверху.
3
draw() — рисует, и только рисует.
4
update() — яблоко продолжает падать.
5
on_mouse_move — PyGame Zero сам зовёт его, когда двигаешь мышью.
Запускаем pgzrun game.py — корзинка едет за курсором. Это магия.
Пиши код прямо перед ребёнком, линию за линией, не копипасти. Чтобы корзинка ехала только по горизонтали — меняем только basket.x. Спроси: «почему бы нам не менять и y?» (получится неестественно — корзинка взлетит к потолку, когда курсор вверху). Это подводка к clamp.
Правило ремесла

draw() — только рисует. update() — только думает.

Nope

Плохо

def draw():
    screen.fill('skyblue')
    apple.y += 2            # логика в draw()!
    basket.draw()
    apple.draw()
OK

Хорошо

def update():
    apple.y += 2            # логика здесь

def draw():
    screen.fill('skyblue')
    basket.draw()
    apple.draw()

Почему это важно. В будущем PyGame Zero сможет пропускать draw() при низком FPS, а update() — нет. Если смешать, игра станет «дрожащей».

Это правило ребёнок услышит и на уроке 37, но здесь имеет смысл повторить — именно сейчас появляются mouse-обработчики, и соблазн написать проверку в draw() велик. Фраза преподавателя из плана: «Не пиши логику в draw() — это место только для рисования».
30-45 мин · самостоятельно · шаг 2

Клик — корзинка улыбается

Python
def reset_basket(): basket.image = 'basket' def on_mouse_down(pos, button): if button == mouse.LEFT: basket.image = 'basket_happy' clock.schedule(reset_basket, 0.3)

Понадобится images/basket_happy.png — корзинка с улыбкой.

Разбираем новое
  • basket.image = 'basket_happy' — меняет картинку у Actor. Без расширения .png.
  • mouse.LEFT — встроенная константа, читается как «левая кнопка».
  • clock.schedule(func, 0.3) — таймер: через 0.3 сек вызовет reset_basket.
Итог: кликнул — корзинка улыбнулась. Через треть секунды сама вернулась. Никаких «зависаний» — поток игры не прерывается.
Первый раз вводим clock.schedule — делай это вместе, не отдавай на самостоятельно. Объясни: «таймер — это заметка на холодильнике, PyGame Zero запомнит и сам вызовет функцию». Подчеркни: функция передаётся БЕЗ скобок (reset_basket, не reset_basket()).
Быстрый словарь таймеров

clock — твой будильник в PyGame Zero

Один раз

clock.schedule(func, sec)

Зовёт функцию один раз через sec секунд.

clock.schedule(reset_basket, 0.3)
Регулярно

clock.schedule_interval(func, sec)

Зовёт каждые sec секунд. Пригодится на уроке 39 для спавна яблок.

clock.schedule_interval(spawn_apple, 1.0)

Приём senior-разработчиков: в schedule всегда передавай имя функции, а не её вызов. reset_basket — это «запиши мне адрес», а reset_basket() — «сделай это прямо сейчас».

Слайд-справочник, можно показать быстро — полноценно использовать interval будем на уроке 39. Но засеять сейчас — хорошо: ребёнок будет готов, и пара вопросов из ДЗ про «подпрыгивание 0.2 сек» опирается на schedule.
45-55 мин · усложнение · шаг 3

Не даём корзинке убежать за экран

Проблема

Если курсор ушёл за край окна, корзинка уедет туда же — и пропадёт.

Решение — «зажим в коридор»
def on_mouse_move(pos):
    x = pos[0]
    x = max(50, min(WIDTH - 50, x))
    basket.x = x

max(50, ...) — не меньше 50. min(WIDTH-50, ...) — не больше WIDTH−50.

Слово дня

clamp

Так называют этот приём «взрослые» программисты. Переводится как «зажим/струбцина». Значит: «возьми число, и если оно меньше минимума — подними до минимума; если больше максимума — опусти до максимума».

Когда встретишь clamp в чужом коде — сразу поймёшь, что это про ограничение диапазона.

Почему именно 50? Половина ширины корзинки — чтобы она не «наполовину вылезала» за экран. Если у тебя корзинка 200 пикселей — используй 100.

Сильный слайд: вводим академический термин (clamp), связываем с конкретной проблемой и даём формулу. Попроси ребёнка сказать вслух формулу своими словами. Если корзинка у ребёнка другая по ширине — поменяйте 50 на половину ширины.

basket = Actor('basket', (400, 520))
apple  = Actor('apple',  (400, 50))

def draw():

    basket.draw()
    apple.draw()

def update():

def reset_basket():

def on_mouse_move(pos):
    x = max(50, min(WIDTH - 50, pos[0]))
    basket.x = x

def on_mouse_down(pos, button):
    if button == mouse.LEFT:
        basket.image = 'basket_happy'
        clock.schedule(reset_basket, 0.3)

Когда можно закрывать урок

Руки
  • Корзинка корректно следует за мышью

  • При клике есть видимая реакция (смена спрайта или подпрыгивание)
Голова

Зелёный флаг

Ребёнок сам предлагает идею — «а давай корзинка ещё…». Это значит, что мышь перестала быть «темой урока» и стала инструментом. Именно этого мы добивались.

Чек-лист — не формальный, а инструмент преподавателя. Если минимум из 4 пунктов не закрыт — лучше чуть сдвинуть урок 39, а 38 зацементировать. Игра строится слоями, провал в мышке сломает коллизии.
