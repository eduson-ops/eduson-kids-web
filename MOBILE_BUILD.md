# Mobile build guide — Eduson Kids

## Overview

This project is a Vite + React SPA wrapped in **Capacitor 8** for native
Android/iOS distribution. Capacitor bundles the built web assets (`dist/`)
into a native shell, giving us: App Store / Play Store presence, push
notifications, camera/filesystem access, and offline support — without a
React Native rewrite. The same codebase also ships as a PWA.

Brand colors are configured in `capacitor.config.ts` (`#0C0533` bg,
`#6B5CE7` accent). Bundle id: `com.edusonkids.app`. `webDir: dist`.

## Assets

Icons are generated from `public/icons/icon-source.svg` via sharp. Regenerate
with `npm run icons` after changing the source SVG.

Outputs (committed):
- `public/icons/icon-192.png`, `icon-512.png`, `icon-maskable-512.png`
- `public/icons/apple-touch-icon.png` (180×180)
- `public/icons/favicon-32.png`, `favicon-16.png`
- `public/manifest.webmanifest` (regenerated by the script too)
- `public/favicon.svg` (source SVG, referenced from `index.html`)

The maskable icon has a 10% safe-zone padding on each side so Android
adaptive masks don't crop the logo.

## Build targets (one codebase → 3 deploys)

Переключение через `VITE_TARGET` env-переменную:

| Target | base | Команда | Куда |
|---|---|---|---|
| `dev` | `/` | `npm run dev` | vite dev-сервер (автоматически) |
| `ghpages` | `/eduson-kids-web/` | `npm run build` | GitHub Pages (default prod) |
| `pwa` | `/eduson-kids-web/` | `VITE_TARGET=pwa npm run build` | Installable PWA |
| `capacitor` | `./` | `VITE_TARGET=capacitor npm run build` | iOS/Android native wrapper |

После любого `capacitor` билда — `npx cap sync` (или `node node_modules/@capacitor/cli/bin/capacitor sync` если `&` в пути).

## Mobile architecture notes

- `src/lib/native.ts` — капабилити-слой: haptic, share, takePhoto, camera, push, preferences, filesystem, status-bar, back-button, keyboard. Всё через dynamic `import('@capacitor/*')` guarded by `Capacitor.isNativePlatform()`.
- `src/components/MobileAppShell.tsx` — обёртка Routes: mounts MobileTabBar, wires hardware back button, keyboard inset → `--kb-height`, app state events (`app:background` / `app:foreground`).
- `src/components/MobileTabBar.tsx` — 5 табов (Главная/Уроки/Студия/Играть/Я), скрыт на `/play/:id`, `/room/:id`, `/studio/:id`, `/learn/lesson/:n/present`.
- `src/components/SmartPythonEditor.tsx` — Monaco на десктопе, CodeMirror 6 на мобилках.
- `src/components/StudioMobileBar.tsx` — мобильный тулбар Studio (Delete/Undo/Mode/Palette/Properties-sheet).
- `src/lib/deviceTier.ts` — low/medium/high probe, shadow-map size, physics timestep, canPostfx.
- `src/lib/audioUnlock.ts` — global first-gesture unlock для Howler/AudioContext + mute on app background.
- `src/hooks/useIsMobile.ts` — `useIsMobile`, `useIsTouch`, `useIsNative`.

## Building for web / PWA

Default flow:

```bash
npm run build          # tsc -b && vite build, emits dist/
npm run preview        # local preview on :4173
```

vite-plugin-pwa picks up `public/icons/*.png` and registers the service
worker. Deployed to GitHub Pages (see `deploy_github_pages.md`).

## Building for Android

Prerequisites:
- Android Studio (latest stable)
- JDK 17 (`java -version` → 17.x). Set `JAVA_HOME`.
- Android SDK 34+ with build-tools 34
- Device in USB-debug mode OR an AVD emulator

Build + open in Android Studio:

```bash
VITE_TARGET=capacitor npm run build
npx cap sync android
npx cap open android
```

In Android Studio: `Run` → pick device. For a release APK/AAB:
`Build → Generate Signed Bundle / APK`.

Signing keystore (one-time, keep it safe, never commit it):

```bash
keytool -genkey -v \
  -keystore eduson-kids-release.keystore \
  -alias eduson \
  -keyalg RSA -keysize 2048 -validity 10000
```

Store the path + passwords in `android/keystore.properties`
(gitignored, шаблон в `android/keystore.properties.example`). Файл уже
прокинут в `android/app/build.gradle` через `signingConfigs.release` —
если `keystore.properties` не существует, release-сборка падает с
понятной ошибкой.

### npm-скрипты для Android (с 2026-04-27)

```
npm run android:sync          # build + cap sync android
npm run android:open          # открыть в Android Studio
npm run android:debug         # gradlew assembleDebug → APK для adb install
npm run android:release-aab   # gradlew bundleRelease → .aab для Play Console
npm run android:logcat        # adb logcat фильтр Capacitor + Console
```

Готовый AAB лежит в `android/app/build/outputs/bundle/release/app-release.aab`.
Загружать в Google Play Console → Internal testing → New release.

`versionCode` нужно бампать перед каждым новым AAB (Play не примет тот же
номер дважды). Текущее значение хранится в `android/app/build.gradle:13`.

## Building for iOS

Prerequisites (macOS only):
- macOS with Xcode 15+ and Command Line Tools
- CocoaPods (`sudo gem install cocoapods` or `brew install cocoapods`)
- Apple Developer account for signing
- iPhone or simulator

On Windows we can scaffold `ios/` (already done via `npx cap add ios`), but
`pod install` and actual builds must happen on macOS.

Flow on macOS:

```bash
VITE_TARGET=capacitor npm run build
npx cap sync ios
cd ios/App && pod install && cd ../..
npx cap open ios
```

In Xcode: select team under **Signing & Capabilities**, pick device,
press Run. For TestFlight: `Product → Archive → Distribute App`.

## Dev live-reload on a real device

```bash
# starts vite dev + injects its URL into the native shell
npx cap run android -l --external
npx cap run ios     -l --external     # macOS only
```

Phone and laptop must be on the same Wi-Fi. `--external` exposes vite on
LAN. Edit React code → see it hot-reload on the device.

## Troubleshooting

- **Gradle sync fails** → confirm JDK 17 (not 21/11). In Android Studio:
  `File → Settings → Build Tools → Gradle → Gradle JDK`. Wipe caches:
  `cd android && ./gradlew --stop && ./gradlew clean`.
- **`SDK location not found`** → create `android/local.properties` with
  `sdk.dir=C:\\Users\\<you>\\AppData\\Local\\Android\\Sdk`.
- **CocoaPods outdated** → `pod repo update && cd ios/App && pod install`.
- **Missing pod command** → `sudo gem install cocoapods`.
- **White screen on launch** → `npx cap sync` after every `npm run build`;
  check `capacitor.config.ts` `server.url` isn't pointing at a dead dev host.
- **Plugin mismatch** → ensure all `@capacitor/*` deps are on the same
  major (currently 8.x). After upgrade: `npx cap sync`.
- **`&` in project path breaks `npx`** → run the CLI directly:
  `node node_modules/@capacitor/cli/bin/capacitor <cmd>`.
