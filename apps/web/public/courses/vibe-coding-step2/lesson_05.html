Что такое VM?

VM (Virtual Machine) — виртуальная копия компьютера, работающая внутри физического сервера в дата-центре.

На одном мощном железе — десятки VM разных клиентов
У каждой свой IP, своя ОС, свои ресурсы
Платишь только за то, что арендуешь
Включается/выключается за 30 секунд

IaaS = Infrastructure as a Service

Аренда железа как услуги. Ты не покупаешь сервер за 100к, ты берёшь его в аренду посекундно.

Теория · 5 мин

SSH — твой телепорт внутрь VM

Как работает SSH-ключ

Приватный ключ (id_ed25519) — хранится только у тебя. Как паспорт. Никому не показывай.
Публичный ключ (id_ed25519.pub) — можно копировать куда угодно. Сервер знает только его.
Сервер шифрует «вызов» публичным ключом — только приватный может его расшифровать.
Никаких паролей в SSH. Только ключи. Пароли брутфорсят за сутки.
~/you @ ноут
# Ключ у нас уже есть с урока 1: $ cat ~/.ssh/id_ed25519.pub ssh-ed25519 AAAAC3Nza... you@laptop # Заходим на VM: $ ssh ubuntu@185.240.112.85 Welcome to Ubuntu 24.04.1 LTS ubuntu@vibe-vm:~$ _

Практика · Шаг 1

Создаём VM за 6 кликов

1Регион: Москва или Санкт-Петербург — чем ближе, тем ниже пинг.
2Образ: Ubuntu 24.04 LTS (свежая, поддерживается 5 лет).
3Конфигурация: 1 vCPU, 1 GB RAM, 10 GB SSD — нам хватит.
4Публичный IP: включить — без него снаружи не зайти.
5SSH-ключ: вставить id_ed25519.pub в поле «Публичные ключи».
6Создать. Через 30–60 секунд — живая VM с IP.

Где взять публичный ключ

PowerShell / Terminal
# Windows: > type $HOME\.ssh\id_ed25519.pub # macOS / Linux: $ cat ~/.ssh/id_ed25519.pub # Копируем всю строку целиком, # от «ssh-ed25519» до e-mail.

Ключ создавался на уроке 1 через ssh-keygen -t ed25519.

Практика · Шаг 2

Первое подключение по SSH

Терминал на твоём ноуте
# Подставляем IP, который дал облачный провайдер: $ ssh ubuntu@185.240.112.85 The authenticity of host '185.240.112.85' can't be established. ED25519 key fingerprint is SHA256:h7x... Are you sure you want to continue connecting (yes/no)? yes Welcome to Ubuntu 24.04.1 LTS (GNU/Linux 6.8.0-45-generic x86_64) * Documentation: https://help.ubuntu.com * Management: https://landscape.canonical.com * Support: https://ubuntu.com/pro ubuntu@vibe-vm:~$ whoami ubuntu ubuntu@vibe-vm:~$ pwd /home/ubuntu

Что означает приглашение ubuntu@vibe-vm:~$

ubuntu — твой пользователь, vibe-vm — имя VM, ~ — домашняя папка, $ — обычные права (не root).

Спросил пароль? Что-то не так

Ключ не подцепился — проверь ~/.ssh/authorized_keys на VM через консоль провайдера.

apt — менеджер пакетов Ubuntu

Как Play Store, только для серверного софта. apt install качает из центральных репозиториев Ubuntu — быстро и безопасно.

Почему sudo?

Установка системного софта — операция администратора. sudo = «выполнить от имени root на одну команду». Безопаснее, чем сидеть рутом.

Флаг -y автоматически отвечает «да» на все вопросы. Удобно на учебной VM. В проде — читаем, что ставим.

Практика · Шаг 5

Клонируем свой репозиторий на VM

Путь А — SSH (правильный)

# 1. На VM генерим новый ключ
ssh-keygen -t ed25519 -C "vibe-vm"
# Enter, Enter, Enter (без парольной фразы)

# 2. Копируем публичный ключ
cat ~/.ssh/id_ed25519.pub

# 3. Идём на GitHub → Settings
#    → SSH and GPG keys → New SSH key
#    Вставляем, сохраняем.

# 4. Клонируем
git clone git@github.com:you/vibe-step2-name.git
cd vibe-step2-name

Путь B — HTTPS + PAT (быстрый)

# 1. GitHub → Settings → Developer settings
#    → Personal access tokens (classic)
#    → Generate new token
#    → scope: repo (минимум)

# 2. Копируем токен (ghp_xxxxxxxx...)

# 3. Клонируем, используем токен как пароль
git clone https://github.com/you/vibe-step2-name.git
# Username: you
# Password: ghp_xxxxxxxx...

cd vibe-step2-name
Токен PAT = пароль. Не коммить его в репо, не показывай другу. Скомпрометировал — сразу отзывай в GitHub.

Практика · Шаг 7 (не забыть!)

Открываем порт 8000 в облаке

По умолчанию облако пускает только SSH (порт 22). Остальное — закрыто.

TW

Timeweb Cloud

Панель VM → вкладка Сеть → Firewall → добавить правило: входящий TCP 8000.

YC

Yandex Cloud

VPC → Security Groups → правило: входящий TCP 8000 с 0.0.0.0/0.

CR

cloud.ru

ЕС2 → Security Groups → Inbound rules → TCP 8000 · Source 0.0.0.0/0.

Если API не открывается в браузере — в 90% случаев проблема тут. Не в коде, не в uvicorn, а в firewall облака. Всегда проверяй первым делом.

Практика · Шаг 8

Скрипт deploy.sh — быстрое обновление

Чтобы не писать 5 команд каждый раз при деплое — объединяем в один скрипт.

#!/bin/bash
# ~/vibe-step2-name/api/deploy.sh
set -e                          # падать на первой ошибке

cd ~/vibe-step2-name/api
echo "→ pull из GitHub..."
git pull

echo "→ обновляем зависимости..."
source .venv/bin/activate
pip install -r requirements.txt

echo "→ перезапускаем uvicorn..."
pkill -f "uvicorn main:app" || true
nohup uvicorn main:app \
  --host 0.0.0.0 --port 8000 \
  > uvicorn.log 2>&1 &

echo "✓ готово. API слушает 8000."
bash
# Даём права на исполнение
chmod +x deploy.sh

# Запускаем
./deploy.sh

# Теперь при каждом пуше с ноута:
#   (ноут) git push
#   (ssh на VM) ./deploy.sh
# код обновился!
bash

Временный фикс для SSH-сессии

nohup ... & = запустить uvicorn в фоне, не убивать при выходе из SSH. Костыль. На следующем уроке переведём на systemd — правильное решение.

Главные мысли урока

SSH — это твой телепорт внутрь удалённого компа. Один раз настроил — забыл, как выглядит пароль.
0.0.0.0 означает «слушаю все интерфейсы». localhost этого не делает.
Никаких паролей в SSH. Только ключи. Пароли брутфорсят за сутки.
Если сессия SSH оборвётся — uvicorn тоже упадёт. На следующем уроке поправим это через systemd.
Прод — это когда твой код работает, даже когда ты спишь.
Всегда смотри, какие порты открыты. Лишний открытый порт — дыра в безопасности.

Урок 6 — systemd. Демонизация процесса

Сегодня у нас проблема:

Закрыл SSH — uvicorn умер
VM перезагрузилась — API не стартанул сам
Процесс упал от ошибки — никто не поднял
Логи пишутся в STDOUT — теряются при рестарте

На следующем уроке:

Autostart при загрузке системы
Restart=always — упал, поднимется сам через 3 сек
journalctl — централизованные логи
Настоящий демон, живущий независимо от терминала
